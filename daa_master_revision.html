📋 Unit 1 — Fundamentals & Asymptotic Notations

Algorithm: Sequence of unambiguous instructions for solving a problem in finite time.

Properties: Finiteness, Definiteness, Input, Output, Effectiveness

Design Strategies: Brute Force, D&C, Decrease & Conquer, Transform & Conquer, Greedy, DP, Backtracking, B&B

Euclid's GCD: gcd(m,n) = gcd(n, m mod n) — O(log n)

Selection Sort:

for i=0 to n-2: find min in A[i..n-1], swap with A[i]

Time: Θ(n²) all cases. Space: O(1). Stable: Yes.

📋 Unit 2 — Brute Force & String Matching

Brute Force: Straightforward approach based directly on problem statement.

Exhaustive Search: Generate all candidates systematically. TSP: Θ((n-1)!), Knapsack: Θ(2n), Assignment: Θ(n!)

Insertion Sort:

for i=1 to n-1: insert A[i] into sorted A[0..i-1]

Best: O(n), Avg/Worst: O(n²), Space: O(1), Stable: Yes

Naive String Matching:

Align pattern at each position, compare character by character. Worst-case: O(mn)

KMP Algorithm (O(m+n)):

Uses prefix function Π to avoid backtracking on text.

Compute-Prefix-Function(p): Π[q] = length of longest proper prefix of p[1..q] that is also suffix.
KMP-Matcher(S,p): scan S left to right, use Π on mismatch.

Rabin-Karp (O(n+m) avg):

Hash pattern and text substrings using rolling hash: ts+1 = d(ts − dm−1T[s+1]) + T[s+m+1]

📋 Unit 3 — Divide & Conquer

Strategy: Divide → Conquer (recursively) → Combine

Merge Sort:

Split array, sort halves, merge → Θ(n log n) all cases, O(n) space, stable

Quick Sort:

Choose pivot, partition, recurse → Θ(n log n) avg, O(n²) worst, O(log n) space, not stable

Binary Search:

l=0, r=n-1; while(l≤r){ m=⌊(l+r)/2⌋; if(K==A[m]) return m; elif(K

Time: Cw(n) = ⌈log₂(n+1)⌉ — e.g., Cw(10⁶) = 20

Closest Pair:


Sort by x, divide, recursive find, check 2d strip. Time: O(n log n)



Quickhull:


Find extreme points, recursively find hull on each side. Worst: O(n²), Avg: O(n)



AVL Trees:


Balance factor = height(L) − height(R) ∈ {−1,0,+1}. Rotations: R, L, LR, RL. Search/Insert/Delete: O(log n)



Linear Time Sorting:


Bucket Sort: O(n+K). Radix Sort: O(d(n+K)). Counting Sort: O(n+K). All stable.

📋 Unit 4 — DP & Greedy

Dynamic Programming:

Solve subproblems once, store results in table, combine to solve larger instance. Overlapping subproblems + optimal substructure.

0/1 Knapsack: V[i,j] = max(V[i−1,j], vᵢ + V[i−1,j−wᵢ]). Time/Space: O(nW)

LCS: len(i,j) = 0/len(i−1,j−1)+1/max(len(i,j−1),len(i−1,j)). Time/Space: O(mn)

Warshall (Transitive Closure): R⁽ᵏ⁾[i,j] = R⁽ᵏ⁻¹⁾[i,j] OR (R⁽ᵏ⁻¹⁾[i,k] AND R⁽ᵏ⁻¹⁾[k,j]). Time: Θ(n³)

Floyd (All-Pairs Shortest Paths): D⁽ᵏ⁾[i,j] = min(D⁽ᵏ⁻¹⁾[i,j], D⁽ᵏ⁻¹⁾[i,k] + D⁽ᵏ⁻¹⁾[k,j]). Time: Θ(n³)

Optimal BST: C[i,j] = min(C[i,k−1] + C[k+1,j]) + Σpₛ. Time: Θ(n³) (→Θ(n²) with monotonicity)

Greedy Technique:

Locally optimal, feasible, irrevocable choices.

Prim's MST: Grow tree adding closest vertex. O(n²) or O(m log n)

Kruskal's MST: Sort edges, add non-cycle edges using Union-Find. O(m log m)

Dijkstra's (SSSP): Similar to Prim, minimize dv + w(v,u). O((V+E) log V). Fails with negative weights.

Huffman Codes: Merge smallest frequency trees. Prefix-free, optimal. Average bits: Σ fᵢ × depthᵢ

📋 Unit 5 — Backtracking & Branch & Bound

Backtracking: Systematic trial-and-error. Try choice → if stuck, undo and try next.

8 Queens: Place queens row by row, check conflicts. ~15,720 placements examined (vs 40,320 permutations).

Subset Sum: Find subset that sums to target. Backtrack with pruning.

Branch & Bound: Systematic enumeration with bounding function to prune suboptimal branches.

TSP (B&B): Lower bound = (sum of 2 nearest distances for each city)/2. Prune branches exceeding current best.

Assignment (B&B): Lower bound = sum of minimum row costs.

Knapsack (B&B): Upper bound = current value + remaining capacity × best value/weight ratio.

Vertex Cover: Subset V' containing at least one endpoint of every edge. NP-Complete.

Bin Packing: Pack items into minimum bins. NP-Hard. First-Fit Decreasing: O(n log n) heuristic.

📋 Unit 6 — Advanced Topics & Complexity

Euclidean Algorithm:

while(y > 0){ r = x mod y; x = y; y = r; } return x;

Time: O(log n)

Modular Arithmetic:

a ≡ b (mod n) if n | (a−b). Properties preserve addition, subtraction, multiplication.

Chinese Remainder Theorem:

For pairwise coprime n₁…nk, system x ≡ aᵢ (mod nᵢ) has unique solution modulo N = n₁·…·nk.

P ≠ NP? Most-believed open problem. If any NPC problem has poly-time solution, then P = NP.

NPC Proof Steps: (1) Prove ∈ NP, (2) Reduce known NPC problem to it.

Reduction Chain: Circuit-SAT → SAT → 3-CNF-SAT → CLIQUE → Vertex Cover → Hamiltonian Cycle → TSP → Subset Sum

🎤 30 Viva Questions (One-Line Answers)

  1. What is an algorithm? — Finite sequence of unambiguous instructions to solve a problem.
  2. What is Big O? — Upper bound: f(n) ≤ c·g(n).
  3. Difference Ω vs Θ? — Ω is lower bound, Θ is tight bound (both upper and lower).
  4. Why Quick Sort preferred over Merge Sort? — In-place (O(log n) space), fast in practice despite O(n²) worst case.
  5. Why Merge Sort preferred for linked lists? — O(1) extra space for linked lists, stable.
  6. When does Binary Search fail? — When array is not sorted or using linked list.
  7. What is stable sorting? — Equal elements retain their relative order.
  8. What is in-place sorting? — Uses O(1) extra memory.
  9. Principle of DP? — Overlapping subproblems + optimal substructure.
  10. DP vs D&C? — DP solves overlapping subproblems once; D&C subproblems are independent.
  11. DP vs Greedy? — DP considers all possibilities; greedy makes locally optimal choice.
  12. Why Dijkstra fails on negative edges? — Once a vertex is finalized, its distance cannot be updated.
  13. Prim vs Dijkstra? — Prim minimizes distance to tree; Dijkstra minimizes distance from source.
  14. KMP advantage? — Never backtracks on text, O(m+n) time.
  15. Prefix function in KMP? — Length of longest proper prefix that is also suffix.
  16. Spurious hit in Rabin-Karp? — Hash matches but string doesn't.
  17. Master Theorem limitation? — f(n) must be polynomial; doesn't apply when f(n) not polynomially comparable.
  18. What is P = NP problem? — Can every problem verifiable in poly time be solved in poly time?
  19. What is NP-Complete? — Hardest problems in NP; if any has poly solution, P=NP.
  20. Prove a problem is NPC? — Show ∈ NP, then reduce known NPC to it in poly time.
  21. AVL vs BST? — AVL is self-balancing, guarantees O(log n) operations.
  22. 2-3 tree property? — All leaves at same depth (perfectly height-balanced).
  23. Radix sort advantage over Quick Sort? — Linear time O(dn) for small d.
  24. Convex hull definition? — Smallest convex polygon containing all points.
  25. Optimality of Huffman? — Greedy choice + optimal substructure proof gives optimal prefix codes.
  26. What is backtracing in DP? — Following table pointers to reconstruct the actual solution.
  27. Branch & Bound vs Backtracking? — B&B uses lower/upper bounds to prune; Backtracking uses constraints.
  28. Fractional vs 0/1 Knapsack? — Fractional is greedy (O(n)), 0/1 is DP (O(nW)).
  29. What is pseudopolynomial? — Time polynomial in numeric value of input, exponential in input length.
  30. What is the Halting Problem? — Undecidable: cannot determine if a program will terminate. It's NP-Hard but not in NP.
