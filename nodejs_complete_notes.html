🚀 Introduction to Node.js Core

What is Node.js?

Node.js is a JavaScript runtime built on Chrome's V8 engine. It allows JavaScript to run outside the browser — on the server. It is event-driven, single-threaded, and uses non-blocking I/O for high scalability.

Key Features

⚡ Event-Driven

Uses an event loop to handle multiple connections concurrently without blocking.

🚫 Non-Blocking I/O

Asynchronous operations don't block the execution thread, enabling high throughput.

🔧 npm Ecosystem

Largest package ecosystem in the world with over 2 million packages.

⚙️ Single-Threaded

Single thread with event loop handles all clients efficiently using callbacks/promises.

Global Objects

JS
// Global objects available everywhere
console.log(global);       // Node's global object
console.log(globalThis);  // ES2020 - works across environments
console.log(__dirname);   // Current directory path
console.log(__filename);  // Current file path
console.log(process);      // Process information
console.log(module);       // Current module info

📦 Modules & Require Core

Concept

Node.js uses the CommonJS module system by default. Each file is treated as a separate module with its own scope. Use module.exports to export and require() to import. ES modules (import/export) are also supported with "type": "module" in package.json.

CommonJS (Default)

JS
// maths.js - Creating a module
const add = (a, b) => a + b;
const PI = 3.14;

module.exports = { add, PI };

// newfile.js - Using the module
const { add, PI } = require('./maths');
console.log(add(5, 10));  // 15

ES Modules

JS
// package.json must have "type": "module"
import express from 'express';
import { promises as fs } from 'fs';

// Named export
export const myFunc = () => {};

// Default export
export default myFunc;
require() import/export module.exports "type": "module" CommonJS

🔔 Event Emitter Core

Concept

The EventEmitter implements the publisher-subscriber pattern. Objects can emit named events and listeners can subscribe to those events. Many Node.js core APIs (like HTTP server, streams) are built on EventEmitter.

Basic Usage

JS
const EventEmitter = require('events');
const emitter = new EventEmitter();

// Listen to event
emitter.on('greet', ({ name, clas }) => {
  console.log(`Hello ${name}, Welcome to ${clas}`);
});

// Emit event
emitter.emit('greet', { name: 'Parveen', clas: 'K23PA' });

Event Methods

MethodDescription
on(event, callback)Register a listener for an event
emit(event, args)Trigger an event with arguments
once(event, callback)Listen once then auto-remove
off(event, callback)Remove a specific listener
removeAllListeners(event)Remove all listeners for an event

Practical Example — Event Tracking System

JS
const fs = require('fs');
const EventEmitter = require('events');
const emitter = new EventEmitter();

let eventCounts = {
  "user-login": 0,
  "user-purchase": 0,
  "user-logout": 0
};

emitter.on('user-login', (username) => {
  eventCounts['user-login']++;
  console.log(`${username} logged in!`);
});

emitter.emit('user-login', 'Parveen');
emitter.emit('summary');

📁 File System (fs) Core

Concept

The fs module provides both synchronous (blocking) and asynchronous (non-blocking) methods for file operations. Always prefer asynchronous methods in production to avoid blocking the event loop.

Three Ways to Use fs

🔴 Synchronous (Blocking)

Blocks the thread until operation completes. Use only in startup/scripts.

🟡 Callback (Async)

Traditional async with error-first callbacks. Non-blocking.

🟢 Promises / Async-Await

Modern approach using fs.promises API. Clean and readable.

1. Synchronous

JS
const fs = require('fs');

fs.writeFileSync('Hello.txt', 'Hello World', 'utf-8');
const data = fs.readFileSync('Hi.txt', 'utf-8');
fs.appendFileSync('Hello.txt', '\nAppended content');
fs.renameSync('Hello.txt', 'Hi.txt');
fs.unlinkSync('file.txt');  // Delete file

2. Asynchronous (Callbacks)

JS
const fs = require('fs');

fs.readFile('Hi.txt', 'utf-8', (err, data) => {
  if (err) console.log(err);
  else console.log(data);
});

fs.writeFile('file.txt', 'content', 'utf-8', (err) => {
  if (err) console.log(err);
  else console.log('File created');
});

fs.appendFile('file.txt', 'New data', (err) => { /* ... */ });
fs.unlink('file.txt', (err) => { /* ... */ });

3. Promises / Async-Await (Recommended)

JS
const fs = require('fs').promises;
// OR: const fs = require('fs/promises');

// Using .then()
fs.readFile('Hi.txt', 'utf-8')
  .then(data => console.log(data))
  .catch(err => console.log(err));

// Using async/await (cleanest)
async function readData() {
  try {
    const data = await fs.readFile('Example.txt', 'utf-8');
    console.log(data);
  } catch (err) {
    console.log(err);
  }
}
readData();

Common fs Methods

MethodDescription
readFile(path, options, callback)Read file contents
writeFile(path, data, options, callback)Write data to file (overwrites)
appendFile(path, data, options, callback)Append data to file
unlink(path, callback)Delete a file
rename(oldPath, newPath, callback)Rename/move a file
mkdir(path, callback)Create a directory
existsSync(path)Check if file exists (sync)

🌊 Streams Core

Concept

Streams allow reading/writing data in chunks instead of loading the entire file into memory. This is crucial for handling large files (e.g., videos, logs, big datasets) efficiently.

Types of Streams

📖 Readable

Read data from a source (file, HTTP request, etc.)

✏️ Writable

Write data to a destination (file, HTTP response, etc.)

🔄 Duplex

Both readable and writable (e.g., TCP socket)

⚡ Transform

Modify data while streaming (e.g., compression)

Read Stream

JS
const fs = require('fs');
const readStream = fs.createReadStream('input.txt', 'utf8');

readStream.on('data', (chunk) => {
  console.log('Received chunk:', chunk);
});
readStream.on('end', () => {
  console.log('Finished reading file.');
});
readStream.on('error', (err) => {
  console.error('Error:', err);
});

Write Stream

JS
const fs = require('fs');
const writeStream = fs.createWriteStream('output.txt');

writeStream.write('Hello Node.js Streams!\n');
writeStream.write('Writing more data...');
writeStream.end('Done writing.');

writeStream.on('finish', () => {
  console.log('All data written successfully.');
});

Pipe — The Elegant Way

JS
// Copy file using pipe (most efficient)
const fs = require('fs');
const readStream = fs.createReadStream('input.txt');
const writeStream = fs.createWriteStream('output.txt');
readStream.pipe(writeStream);

Transform Stream

JS
const { Transform } = require('stream');
const upperCase = new Transform({
  transform(chunk, encoding, callback) {
    this.push(chunk.toString().toUpperCase());
    callback();
  }
});
fs.createReadStream('input.txt')
  .pipe(upperCase)
  .pipe(fs.createWriteStream('output.txt'));

🗜️ Zlib Compression Core

Concept

The zlib module provides compression/decompression using Gzip, Deflate, and other algorithms. It supports both stream-based and buffer-based operations.

Stream-based (Pipe Method)

JS
const fs = require('fs');
const zlib = require('zlib');

// Compress input.txt → input.txt.gz
fs.createReadStream('input.txt')
  .pipe(zlib.createGzip())
  .pipe(fs.createWriteStream('input.txt.gz'))
  .on('finish', () => console.log('Compressed!'));

// Decompress input.txt.gz → output.txt
fs.createReadStream('input.txt.gz')
  .pipe(zlib.createGunzip())
  .pipe(fs.createWriteStream('output.txt'))
  .on('finish', () => console.log('Decompressed!'));

Buffer-based

JS
const zlib = require('zlib');

// Async
zlib.gzip('Hello Node.js!', (err, compressed) => {
  zlib.gunzip(compressed, (err, decompressed) => {
    console.log(decompressed.toString());
  });
});

// Sync
const comp = zlib.gzipSync('Hello!');
const decomp = zlib.gunzipSync(comp);
console.log(decomp.toString());

🌐 HTTP Module Important

Concept

The http module lets you create HTTP servers and clients without any external framework. The server is an EventEmitter that listens for requests and sends responses.

Basic HTTP Server

JS
const http = require('http');

const server = http.createServer((req, res) => {
  // req = request object (url, method, headers)
  // res = response object (send data back)
  
  if (req.url === '/') {
    res.writeHead(200, { 'Content-Type': 'text/html' });
    res.write('<h1>Hello World</h1>');
    res.end();
  } else if (req.url === '/about') {
    res.end('About Page');
  } else {
    res.writeHead(404);
    res.end('Not Found');
  }
});

server.listen(3000, () => {
  console.log('Server running on port 3000');
});

Working with Headers

JS
// Set headers
res.setHeader('Content-Type', 'text/html');
res.setHeader('Connection', 'close');

// Get headers
const contentType = res.getHeader('Content-Type');

// Write response head (status, message, headers)
res.writeHead(200, 'OK', { 'Content-Type': 'text/plain' });

// Read request headers
console.log(req.headers.host);      // Host header
console.log(req.headers['accept-language']);

Handling Form Data (POST)

JS
const http = require('http');
const querystring = require('querystring');

http.createServer((req, res) => {
  if (req.method === 'POST') {
    let body = [];
    req.on('data', chunk => body.push(chunk));
    req.on('end', () => {
      const data = querystring.parse(Buffer.concat(body).toString());
      console.log(data.name, data.email);
      res.end('Data received');
    });
  }
}).listen(3000);

Reading HTML Files and Serving

JS
const fs = require('fs');
fs.readFile('Html/first.html', 'utf8', (err, data) => {
  if (err) {
    res.writeHead(404);
    res.end('File not found');
    return;
  }
  res.writeHead(200, { 'Content-Type': 'text/html' });
  res.write(data);
  res.end();
});

⚡ Express.js Important

What is Express?

Express.js is a minimal, flexible web framework for Node.js. It provides a robust set of features for web and mobile applications, including routing, middleware, template engines, and static file serving.

Basic Setup

JS
import express from 'express';
const app = express();

// Middleware to parse request bodies
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// Route
app.get('/', (req, res) => {
  res.send('Hello Express!');
});

app.listen(3000, () => console.log('Server running'));

HTTP Methods (CRUD)

JS
// GET - Retrieve data
app.get('/users', (req, res) => {
  res.json(users);
});

// GET by ID
app.get('/users/:id', (req, res) => {
  const user = users.find(u => u.id === req.params.id);
  if (!user) return res.status(404).json({ error: 'Not found' });
  res.json(user);
});

// POST - Create data
app.post('/users', (req, res) => {
  const { name, email } = req.body;
  const newUser = { id: Date.now(), name, email };
  users.push(newUser);
  res.status(201).json(newUser);
});

// PUT - Update data
app.put('/users/:id', (req, res) => {
  const idx = users.findIndex(u => u.id === req.params.id);
  if (idx === -1) return res.status(404).json({ error: 'Not found' });
  users[idx] = { ...users[idx], ...req.body };
  res.json(users[idx]);
});

// DELETE - Remove data
app.delete('/users/:id', (req, res) => {
  const idx = users.findIndex(u => u.id === req.params.id);
  if (idx === -1) return res.status(404).json({ error: 'Not found' });
  users.splice(idx, 1);
  res.json({ message: 'Deleted' });
});

URL Parameters & Query Strings

JS
// Route parameters (:id, :name)
app.get('/user/:id/:name', (req, res) => {
  const { id, name } = req.params;
  res.send(`Welcome ${name}, ID: ${id}`);
});

// Query strings (?keyword=hello)
app.get('/search', (req, res) => {
  const keyword = req.query.keyword;
  res.send(`Searching for ${keyword}`);
});

Regular Expression Routes

JS
// ? makes previous char optional (ab?cd → /acd or /abcd)
app.get(/ab?cd/, (req, res) => res.send('Matched /acd or /abcd'));

// * matches any chars (ab*cd → /abcd, /abbcd, etc.)
app.get(/ab*cd/, (req, res) => res.send('Matched ab*cd'));

// + means 1 or more
app.get(/ab+cd/, (req, res) => res.send('Matched /abcd, /abbcd'));

// Start with /a
app.get(/^\/a/, (req, res) => res.send('Starts with a'));

// Match only numbers
app.get(/^\/[0-9]+$/, (req, res) => res.send('Only numbers'));

// Wildcard / fallback (catch-all)
app.get(/.*/, (req, res) => res.send('Catch-all route'));

// Redirect
app.get('/about', (req, res) => res.redirect('/user'));

Serving Static Files

JS
// Serve files from 'public' directory
app.use(express.static('public'));

// Or send specific file
app.get('/form', (req, res) => {
  res.sendFile('form.html', { root: 'public' });
});

Sending Responses

JS
res.send('Hello World');                // Text/HTML
res.json({ name: 'John' });              // JSON
res.status(201).json(newUser);          // With status code
res.redirect('/home');                   // Redirect
res.sendFile('file.html', { root: '.' }); // File
res.download('path/to/file.pdf');         // Download

File-based CRUD with JSON Storage

JS
import { promises as fs } from 'fs';

async function readUsers() {
  try {
    const data = await fs.readFile('data/users.json', 'utf-8');
    return JSON.parse(data);
  } catch { return []; }
}

async function writeUsers(users) {
  await fs.writeFile('data/users.json', JSON.stringify(users, null, 2));
}

function nextId(users) {
  return users.length ? Math.max(...users.map(u => u.id)) + 1 : 1;
}

// CREATE
app.post('/users', async (req, res) => {
  const users = await readUsers();
  const user = { id: nextId(users), ...req.body };
  users.push(user);
  await writeUsers(users);
  res.status(201).json(user);
});

// READ all
app.get('/users', async (req, res) => {
  const users = await readUsers();
  res.json(users);
});

// UPDATE
app.put('/users/:id', async (req, res) => {
  let users = await readUsers();
  const idx = users.findIndex(u => u.id === Number(req.params.id));
  if (idx === -1) return res.status(404).json({ error: 'Not found' });
  users[idx] = { ...users[idx], ...req.body };
  await writeUsers(users);
  res.json(users[idx]);
});

// DELETE
app.delete('/users/:id', async (req, res) => {
  let users = await readUsers();
  users = users.filter(u => u.id !== Number(req.params.id));
  await writeUsers(users);
  res.json({ message: 'Deleted' });
});

🔗 Middleware Express

What is Middleware?

Middleware are functions that have access to the request object (req), response object (res), and the next middleware function. They can execute code, modify req/res, end the request cycle, or call the next middleware.

Types of Middleware

📋 Application-Level

Applied with app.use() — runs for every request.

🗺️ Router-Level

Applied to specific routes with router.use().

⚠️ Error-Handling

4-parameter function (err, req, res, next). MUST be last.

📦 Built-in

express.json(), express.urlencoded(), express.static()

📎 Third-Party

Morgan (logging), cors, helmet, cookie-parser

Basic Middleware Examples

JS
// 1. Application-level middleware
app.use(express.json());  // Parse JSON bodies
app.use(express.urlencoded({ extended: true }));  // Parse form data

// 2. Custom logging middleware
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();  // MUST call next() or send response
});

// 3. Route-specific middleware (age check)
function ageCheck(req, res, next) {
  const age = Number(req.query.age);
  if (age < 18) {
    return res.send('Not allowed (under 18)');
  }
  next();
}

// Apply route-specific middleware
app.get('/user', ageCheck, (req, res) => {
  res.send('Welcome user');
});

// 4. Error-handling middleware (MUST be last)
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something broke!');
});

// 5. Third-party logging with Morgan
import morgan from 'morgan';
app.use(morgan('dev'));

🍪 Cookies & Sessions Express

Concept

Cookies are small pieces of data stored on the client's browser. Sessions store data on the server and use a session ID cookie to identify users. Express-session and cookie-session are two popular session middleware.

Cookie-Parser

JS
import express from 'express';
import cookieParser from 'cookie-parser';

const app = express();
app.use(cookieParser());

// Set cookie
app.get('/setcookie', (req, res) => {
  res.cookie('username', 'Rabaab', {
    maxAge: 1000 * 60,  // 1 minute
    httpOnly: true       // Not accessible via JS
  });
  res.send('Cookie set!');
});

// Get cookie
app.get('/getcookie', (req, res) => {
  res.send('Cookie: ' + req.cookies.username);
});

// Clear cookie
app.get('/clearcookie', (req, res) => {
  res.clearCookie('username');
  res.send('Cookie cleared!');
});

Cookie Count (Visit Tracker)

JS
app.get('/home', (req, res) => {
  let count = 0;
  if (req.cookies.homeVisit) {
    count = parseInt(req.cookies.homeVisit) + 1;
  } else {
    count = 1;
  }
  res.cookie('homeVisit', count, {
    maxAge: 24 * 60 * 60 * 1000,  // 1 day
    httpOnly: true
  });
  res.send(`You visited ${count} times`);
});

Express-Session

JS
import express from 'express';
import session from 'express-session';

app.use(session({

🗄️ MongoDB Database

Concept

MongoDB is a NoSQL database that stores data in flexible, JSON-like documents. Node.js connects to MongoDB using the native mongodb driver or the mongoose ODM.

Native MongoDB Driver

JS
import { MongoClient } from 'mongodb';

const url = 'mongodb://localhost:27017';
const client = new MongoClient(url);

async function main() {
  try {
    await client.connect();
    const db = client.db('myDatabase');
    const collection = db.collection('users');

    // CREATE
    const result = await collection.insertOne({ name: 'Rohan', age: 22 });
    console.log('Inserted:', result.insertedId);
  } finally {
    await client.close();
  }
}
main();

MongoDB CRUD with Express

JS
import { MongoClient, ObjectId } from 'mongodb';

let collection;

// READ all
app.get('/users', async (req, res) => {
  const users = await collection.find().sort({ createdAt: -1 }).toArray();
  res.json(users);
});

// CREATE
app.post('/users', async (req, res) => {
  const doc = { name: req.body.name, email: req.body.email, createdAt: new Date() };
  const result = await collection.insertOne(doc);
  res.status(201).json({ ...doc, _id: result.insertedId });
});

// UPDATE
app.put('/users/:id', async (req, res) => {
  const result = await collection.findOneAndUpdate(
    { _id: new ObjectId(req.params.id) },
    { $set: { name: req.body.name, email: req.body.email, updatedAt: new Date() } },
    { returnDocument: 'after' }
  );
  res.json(result.value);
});

// DELETE
app.delete('/users/:id', async (req, res) => {
  const result = await collection.deleteOne({ _id: new ObjectId(req.params.id) });

Concept

Socket programming enables real-time, bidirectional communication between clients and servers. Node.js supports TCP sockets via the net module, WebSocket via the ws library, and Socket.IO for cross-transport real-time communication.

💬 Socket.IO Real-time

Concept

Socket.IO enables real-time, bidirectional, event-based communication. It uses WebSocket as transport and falls back to HTTP long-polling. It's the most popular library for building chat apps, live notifications, and collaborative tools.

Server Setup

JS
import express from 'express';
import http from 'http';
import { Server } from 'socket.io';

const app = express();
const server = http.createServer(app);
const io = new Server(server, {
  cors: { origin: '*' }
});

// Serve static files
app.use(express.static('public'));

io.on('connection', (socket) => {
  console.log('A user connected:', socket.id);

  // Receive from client
  socket.on('user-message', (message) => {
    console.log('Message:', message);
    
    // Broadcast to ALL connected clients
    io.emit('server-message', message);
  });

  socket.on('disconnect', () => {
    console.log('User disconnected');
  });
});

server.listen(4000, () => console.log('Server running on port 4000'));

Client Setup

🎤 Viva Questions & Answers Important

📌 Tips for Viva

Focus on the "why" not just "what". Understand concepts like event loop, non-blocking I/O, streams, and middleware. Practice writing code from scratch for CRUD operations.

Q1: What is Node.js?
Answer: Node.js is a JavaScript runtime built on Chrome's V8 engine that allows JavaScript to run outside the browser. It uses an event-driven, non-blocking I/O model making it lightweight and efficient for data-intensive real-time applications across distributed devices.
Q2: Difference between synchronous and asynchronous code?
Answer: Synchronous code executes line by line, blocking the thread until each operation completes. Asynchronous code doesn't wait — it uses callbacks, promises, or async-await to handle results when they're ready without blocking.
Q3: What is Express.js?
Answer: Express.js is a minimal, flexible web framework for Node.js that simplifies building web applications and APIs. It provides routing, middleware, request/response handling, and static file serving.
Q4: Difference between app.use() and app.get()?
Answer: app.use() is middleware that runs for ALL requests (GET, POST, etc.) matching the path. app.get() is a route handler that only responds to GET requests for a specific path.
Q5: What are middlewares in Express?
Answer: Middleware functions have access to request, response, and the next middleware function. They can execute code, modify request/response, end the request cycle, or call next middleware. Types include application-level, router-level, error-handling, and third-party.
Q6: What is req.params vs req.query?
Answer: req.params captures route parameters defined with :param in the URL path (e.g., /users/:id). req.query captures query string parameters (e.g., ?keyword=hello).
Q7: How to handle errors in Express?
Answer: Using error-handling middleware with 4 parameters (err, req, res, next). It must be placed after all other routes. Call next(err) in route handlers to pass errors to this middleware.
Q8: What are streams in Node.js?
Answer: Streams allow reading/writing data in chunks instead of loading everything into memory. Types: Readable, Writable, Duplex (both), Transform (modify while streaming). Efficient for large files and network data.
Q9: What is EventEmitter?
Answer: EventEmitter implements the publisher-subscriber pattern. Objects can emit named events and listeners subscribe to them. Methods: on(), emit(), once(), off(). Many Node.js APIs (HTTP, streams) are built on it.
Q10: Difference between require() and import?
Answer: require() is CommonJS (default in Node.js), synchronous, can be used anywhere. import is ES6 module syntax, asynchronous, requires "type": "module" in package.json, and must be at the top of the file.
Q11: How to create a REST API?
Answer: Use HTTP methods: GET (read), POST (create, 201 status), PUT (update), DELETE (remove). Use Express routes with proper status codes, JSON parsing, and data validation.
Q12: What is CORS?
Answer: Cross-Origin Resource Sharing allows requests from different domains. Use the cors npm package. Without CORS, browsers block cross-origin requests for security.
Q13: How to validate user input?
Answer: Use express-validator for server-side validation. Check required fields, data types, formats (email), and lengths. Return 400 status with descriptive error messages for invalid data.
Q14: What is Socket.IO?
Answer: Socket.IO enables real-time, bidirectional communication between client and server. It uses WebSocket primarily with HTTP long-polling fallback. Used for chat apps, live notifications, collaborative tools, and real-time dashboards.
Q15: What is package.json?
Answer: Configuration file containing project metadata (name, version), dependencies, devDependencies, scripts, and other settings. It's essential for managing Node.js projects.
Q16: What is nodemon?
Answer: A development tool that automatically restarts the Node.js application when file changes are detected. Installed as a dev dependency with npm install -D nodemon.
Q17: Difference between PUT and PATCH?
Answer: PUT replaces the entire resource (full update). PATCH applies partial modifications. Use PUT when sending all fields, PATCH when sending only changed fields.
Q18: What is callback hell and how to avoid it?
Answer: Deeply nested callbacks making code hard to read and maintain. Avoid using Promises (.then().catch()) or async/await (cleanest approach) instead of nested callbacks.
Q19: What is the Event Loop?
Answer: The event loop is what allows Node.js to perform non-blocking I/O operations despite being single-threaded. It offloads operations to the kernel (via libuv) and processes callbacks when operations complete.
Q20: Difference between MongoDB and Mongoose?
Answer: MongoDB is the database. Mongoose is an ODM (Object Document Mapper) that provides schema definition, validation, query building, and middleware on top of MongoDB. Mongoose adds structure and validation to MongoDB operations.
