Unit 1: Fundamentals of Algorithms & Data Structures (Click to expand)

What is an Algorithm?

Definition: An algorithm is a sequence of unambiguous instructions for solving a problem, i.e., for obtaining a required output for any legitimate input in a finite amount of time.

Properties of an Algorithm:

  • Finiteness: Terminates after a finite number of steps
  • Definiteness: Rigorously and unambiguously specified
  • Input: Zero or more quantities are externally supplied
  • Output: At least one quantity is produced
  • Effectiveness: Steps are sufficiently basic

Algorithm Design Strategies:

  • Brute Force
  • Divide and Conquer
  • Decrease and Conquer
  • Transform and Conquer
  • Greedy Approach
  • Dynamic Programming
  • Backtracking and Branch-and-Bound
  • Space and Time Tradeoffs

Important Problem Types:

  • Sorting, Searching, String Processing
  • Graph Problems, Combinatorial Problems
  • Geometric Problems, Numerical Problems

Euclid's Algorithm (GCD)

Problem: Find gcd(m,n), the greatest common divisor of two nonnegative, not both zero integers m and n.

Principle: gcd(m,n) = gcd(n, m mod n)

Pseudocode:

while n ≠ 0 do
    r ← m mod n
    m ← n
    n ← r
return m

Dry Run:

gcd(60,24) = gcd(24,12) = gcd(12,0) = 12

Complexity:

Time: O(log n) — logarithmic

Alternative Methods:

  • Consecutive integer checking: O(n)
  • Middle-school procedure: O(√n)
💡 Intuitive Analogy: Like measuring two sticks by repeatedly replacing the longer with the remainder — you eventually get the longest common measure.
// C++ implementation of Euclid's algorithm (iterative)
int gcd(int m, int n) {
    while (n != 0) {
        int r = m % n;
        m = n;
        n = r;
    }
    return m;
}
// Time complexity: O(log min(m,n))

Fundamental Data Structures

Linear Data Structures:

  • Array: Fixed-length, contiguous memory, direct access — O(1) access, O(n) insert/delete
  • Linked List: Dynamic length, arbitrary memory, access by following links — Singly/Doubly linked
  • Stack: LIFO — Push/Pop operations
  • Queue: FIFO — Enqueue/Dequeue operations
  • Priority Queue / Heap: Elements with keys/priorities — find/delete max, insert

Graphs:

G = <V,E> — finite set of vertices V and edges E

  • Undirected/Directed graphs
  • Complete graph K|V|: every pair connected
  • Representation: Adjacency matrix (n×n), Adjacency linked lists
  • Weighted graphs: Numbers assigned to edges
  • Path: Sequence of adjacent vertices; Simple path: distinct edges
  • Connected graph: Path between every pair; Cycle: simple path starting & ending same vertex
  • Tree: Connected acyclic graph; |E| = |V| - 1

Binary Trees:

  • Binary Search Tree: Left subtree < parent < right subtree
  • Height: ⌊log₂n⌋ ≤ h ≤ n−1
  • Tree properties: Depth, Height, Ancestors, Descendants, Leaves

Asymptotic Notations

The efficiency analysis framework concentrates on the order of growth of an algorithm's basic operation count.

Big O Notation (Upper Bound):

f(n) = O(g(n)) if ∃ constants c > 0, n₀ > 0 such that 0 ≤ f(n) ≤ c·g(n) for all n ≥ n₀

"f(n) grows no faster than g(n)"

Big Omega Notation (Lower Bound):

f(n) = Ω(g(n)) if ∃ constants c > 0, n₀ > 0 such that 0 ≤ c·g(n) ≤ f(n) for all n ≥ n₀

"f(n) grows at least as fast as g(n)"

Big Theta Notation (Tight Bound):

f(n) = Θ(g(n)) if f(n) = O(g(n)) AND f(n) = Ω(g(n))

"f(n) grows at the same rate as g(n)"

Small o and ω Notations:

  • o(g(n)): For any c > 0, ∃ n₀ > 0 such that 0 ≤ f(n) < c·g(n) — like < (strictly less)
  • ω(g(n)): For any c > 0, ∃ n₀ > 0 such that 0 ≤ c·g(n) < f(n) — like > (strictly greater)

Properties:

  • f(n) ∈ O(f(n)) — Reflexivity
  • f(n) ∈ O(g(n)) iff g(n) ∈ Ω(f(n)) — Transpose symmetry
  • If f(n) ∈ O(g(n)) and g(n) ∈ O(h(n)), then f(n) ∈ O(h(n)) — Transitivity
  • If f₁(n) ∈ O(g₁(n)) and f₂(n) ∈ O(g₂(n)), then f₁(n) + f₂(n) ∈ O(max{g₁(n), g₂(n)})

Order of Growth (by limits):

limn→∞ T(n)/g(n) = 0  ⇒ T(n) < g(n)
                         = c > 0 ⇒ T(n) = g(n)
                         = ∞    ⇒ T(n) > g(n)

Orders of Growth Hierarchy:

1 < log n < nα (α>0) < an < n! < nn

Basic Efficiency Classes:

L'Hôpital's Rule & Stirling's Formula:

If lim f(n) = lim g(n) = ∞, then lim f(n)/g(n) = lim f'(n)/g'(n)

Stirling's formula: n! ≈ √(2πn) · (n/e)n

💡 Intuitive Analogy: Big O is like a speed limit (you never go faster), Omega is a minimum speed, Theta is a constant speed — they describe how an algorithm's time grows as input size grows.

Selection Sort

Algorithm: Scan array to find smallest element, swap with first element. Repeat for remaining elements.

Algorithm SelectionSort(A[0..n-1])
for i ← 0 to n – 2 do
    min ← i
    for j ← i + 1 to n – 1 do
        if A[j] < A[min]
            min ← j
    swap A[i] and A[min]

Complexity:

  • Time: Θ(n²) — all cases
  • Space: Θ(1) — in place
  • Stable: Yes
💡 Intuitive Analogy: Like picking the smallest card from a hand and placing it at the front, then repeating for the rest.
// C++ selection sort
void selectionSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        int minIdx = i;
        for (int j = i+1; j < n; j++)
            if (arr[j] < arr[minIdx]) minIdx = j;
        std::swap(arr[i], arr[minIdx]);
    }
}

Unit 2: Brute Force & String Matching (Click to expand)

Brute Force Approach

Definition: A straightforward approach, usually based directly on the problem's statement and definitions.

Examples:

  • Computing an (a > 0, n ≥ 0)
  • Computing n!
  • Multiplying two matrices
  • Searching for a key in a list

Strengths:

  • Wide applicability, simplicity
  • Yields reasonable algorithms for some important problems

Weaknesses:

  • Rarely yields efficient algorithms
  • Some brute-force algorithms are unacceptably slow

Brute-Force String Matching (Naive Algorithm)

Problem: Find pattern P[1..m] in text T[1..n] where n > m.

Algorithm:

Step 1: Align pattern at beginning of text
Step 2: Compare each character of pattern to text
Step 3: If mismatch, realign pattern one position right & repeat

Complexity:

Worst-case: Θ(mn) comparisons

Example: Pattern = 001011, Text = 10010101101001100101111010

💡 Intuitive Analogy: Like sliding a stencil over a wall, checking each position character by character — if it doesn't match, shift by one.
// Brute-force string matching in C++
int naiveSearch(const string& text, const string& pat) {
    int n = text.size(), m = pat.size();
    for (int i = 0; i <= n - m; i++) {
        int j = 0;
        while (j < m && text[i+j] == pat[j]) j++;
        if (j == m) return i; // match found
    }
    return -1;
}

Insertion Sort

Algorithm: Build sorted array one element at a time by repeatedly swapping element i with the one to its left if smaller.

Algorithm InsertionSort(A[0..n-1])
for i ← 1 to n-1 do
    v ← A[i]
    j ← i-1
    while j ≥ 0 and A[j] > v do
        A[j+1] ← A[j]
        j ← j-1
    A[j+1] ← v

Dry Run (sorting: 7, 3, 2, 5):

Initially: [7, 3, 2, 5]
i=1: [3, 7, 2, 5] — 3 moves left of 7
i=2: [2, 3, 7, 5] — 2 moves to front
i=3: [2, 3, 5, 7] — 5 inserted correctly

Complexity:

  • Best: O(n) — already sorted
  • Average: O(n²)
  • Worst: O(n²) — reverse sorted
  • Space: O(1) — in place
  • Stable: Yes
💡 Intuitive Analogy: Like sorting cards in your hand — pick a card and insert it into the correct position among the already sorted cards on the left.
// C++ insertion sort
void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j+1] = arr[j];
            j--;
        }
        arr[j+1] = key;
    }
}

Exhaustive Search

Definition: A brute force solution to problems involving search for an element with a special property, usually among combinatorial objects (permutations, combinations, subsets).

Traveling Salesman Problem (TSP):

Given n cities with known distances, find shortest tour visiting all cities exactly once and returning to start.

Efficiency: Θ((n-1)!) — generates all permutations

Knapsack Problem:

Given n items with weights wᵢ and values vᵢ, and capacity W, find most valuable subset that fits.

Efficiency: Θ(2n) — generates all subsets

Assignment Problem:

n people assigned to n jobs, one per person. Minimize total cost C[i,j].

Efficiency: Θ(n!) — generates all permutations

Better alternative: Hungarian method — O(n³)

Knuth-Morris-Pratt (KMP) Algorithm

Purpose: Linear-time string matching algorithm that avoids backtracking on the text.

Key insight: When a mismatch occurs, use knowledge of pattern itself to determine next alignment — never re-examine matched characters in text.

Prefix Function Π:

Compute-Prefix-Function(p)
1  m ← length[p]
2  Π[1] ← 0
3  k ← 0
4  for q ← 2 to m
5      do while k > 0 and p[k+1] ≠ p[q]
6          do k ← Π[k]
7      if p[k+1] = p[q]
8          then k ← k + 1
9      Π[q] ← k
10 return Π

KMP Matcher:

KMP-Matcher(S,p)
1  n ← length[S]
2  m ← length[p]
3  Π ← Compute-Prefix-Function(p)
4  q ← 0
5  for i ← 1 to n
6      do while q > 0 and p[q+1] ≠ S[i]
7          do q ← Π[q]
8      if p[q+1] = S[i]
9          then q ← q + 1
10     if q = m
11         then print "Pattern occurs with shift" i – m
12         q ← Π[q]

Complexity:

  • Prefix function: Θ(m)
  • Matching: Θ(n)
  • Total: O(m + n)
💡 Intuitive Analogy: Like remembering the parts of the pattern that overlap — when a mismatch happens, you don't restart from scratch; you reuse what you already know.
// Compute prefix function (LPS) for KMP
void computeLPS(const string& pat, int lps[]) {
    int len = 0, i = 1;
    lps[0] = 0;
    while (i < pat.size()) {
        if (pat[i] == pat[len]) lps[i++] = ++len;
        else if (len) len = lps[len-1];
        else lps[i++] = 0;
    }
}
// KMP search: O(n+m)
int kmpSearch(const string& text, const string& pat) {
    int n = text.size(), m = pat.size();
    int lps[m];
    computeLPS(pat, lps);
    int i = 0, j = 0;
    while (i < n) {
        if (pat[j] == text[i]) { i++; j++; }
        if (j == m) { return i - j; } // match at i-j
        else if (i < n && pat[j] != text[i]) {
            if (j) j = lps[j-1];
            else i++;
        }
    }
    return -1;
}

Rabin-Karp Algorithm

Idea: Use hashing to find pattern matches. Treat each character as a digit, compute hash of pattern and each text substring.

Key Formulas:

p = P[m] + d(P[m-1] + d(P[m-2] + ... + d(P[2] + dP[1])))  // Horner's rule
ts+1 = d(ts - dm-1T[s+1]) + T[s+m+1]

Complexity:

  • Worst-case: O((n-m+1)m)
  • Average-case: O(n+m) (with good hash)

Note: Spurious hits occur when hash matches but strings don't — use modulo to keep numbers manageable.

💡 Intuitive Analogy: Like comparing fingerprints (hashes) — if fingerprints match, then you do a detailed check; otherwise skip.

Unit 3: Divide & Conquer (Click to expand)

Divide & Conquer Strategy

Principle:

  1. Divide: Split problem into two or more smaller subproblems
  2. Conquer: Solve smaller instances recursively
  3. Combine: Combine solutions to get original solution

Examples:

  • Merge Sort, Quick Sort
  • Binary Tree Traversals
  • Binary Search
  • Large Integer Multiplication
  • Strassen's Matrix Multiplication
  • Closest Pair, Convex Hull
💡 Intuitive Analogy: Like breaking a big task into smaller, independent tasks, solving each, and then combining the results — think of dividing a large document into paragraphs, editing each, then merging.

Merge Sort

Algorithm:

  1. Split array A[0..n-1] into about equal halves
  2. Sort each half recursively
  3. Merge sorted halves
Algorithm MergeSort(A[0..n-1])
if n > 1
    copy A[0..⌊n/2⌋-1] to B[0..⌊n/2⌋-1]
    copy A[⌊n/2⌋..n-1] to C[0..⌈n/2⌉-1]
    MergeSort(B)
    MergeSort(C)
    Merge(B, C, A)

Complexity:

  • Best: Θ(n log n)
  • Average: Θ(n log n)
  • Worst: Θ(n log n)
  • Space: O(n) — not in place
  • Stable: Yes
💡 Intuitive Analogy: Like splitting a deck of cards into two halves, sorting each half separately, then merging the two sorted halves together.
// Merge two sorted subarrays
void merge(int arr[], int l, int m, int r) {
    int n1 = m-l+1, n2 = r-m;
    int L[n1], R[n2];
    for (int i=0; i<n1; i++) L[i] = arr[l+i];
    for (int j=0; j<n2; j++) R[j] = arr[m+1+j];
    int i=0, j=0, k=l;
    while (i<n1 && j<n2) arr[k++] = (L[i] <= R[j]) ? L[i++] : R[j++];
    while (i<n1) arr[k++] = L[i++];
    while (j<n2) arr[k++] = R[j++];
}
void mergeSort(int arr[], int l, int r) {
    if (l < r) {
        int m = l + (r-l)/2;
        mergeSort(arr, l, m);
        mergeSort(arr, m+1, r);
        merge(arr, l, m, r);
    }
}

Quick Sort

Algorithm:

  1. Select a pivot element
  2. Partition array so elements < pivot are left, > pivot are right
  3. Recursively sort left and right subarrays
Algorithm QuickSort(A[l..r])
if l < r
    s ← Partition(A[l..r])  // s is split position
    QuickSort(A[l..s-1])
    QuickSort(A[s+1..r])

Complexity:

  • Best: Θ(n log n) — balanced partition
  • Average: Θ(n log n)
  • Worst: Θ(n²) — already sorted array
  • Space: O(log n) — in place
  • Stable: No
💡 Intuitive Analogy: Like choosing a "pivot" person in a line, then arranging everyone shorter on the left and taller on the right, and recursively doing the same on each side.
// Partition for Quick Sort
int partition(int arr[], int low, int high) {
    int pivot = arr[high];
    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (arr[j] < pivot) std::swap(arr[++i], arr[j]);
    }
    std::swap(arr[i+1], arr[high]);
    return i+1;
}
void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi-1);
        quickSort(arr, pi+1, high);
    }
}

Binary Search

Algorithm: Very efficient algorithm for searching in sorted array.

l ← 0; r ← n-1
while l ≤ r do
    m ← ⌊(l+r)/2⌋
    if K = A[m] return m
    else if K < A[m] r ← m-1
    else l ← m+1
return -1

Complexity:

  • Worst-case: Cw(n) = ⌈log₂(n+1)⌉
  • Cw(10⁶) = 20 — VERY fast!
  • Limitation: Must be sorted array (not linked list)
💡 Intuitive Analogy: Like guessing a number in a range: each time you guess the middle, and if it's too high, you discard the upper half; too low, discard the lower half.
// Binary Search (iterative)
int binarySearch(int arr[], int n, int key) {
    int l = 0, r = n-1;
    while (l <= r) {
        int m = l + (r-l)/2;
        if (arr[m] == key) return m;
        if (arr[m] < key) l = m+1;
        else r = m-1;
    }
    return -1;
}

Strassen's Matrix Multiplication

Idea: Multiply two 2×2 matrices using 7 multiplications instead of 8.

m₁ = (a₀₀ + a₁₁)(b₀₀ + b₁₁)
m₂ = (a₁₀ + a₁₁)b₀₀
m₃ = a₀₀(b₀₁ - b₁₁)
m₄ = a₁₁(b₁₀ - b₀₀)
m₅ = (a₀₀ + a₀₁)b₁₁
m₆ = (a₁₀ - a₀₀)(b₀₀ + b₀₁)
m₇ = (a₀₁ - a₁₁)(b₁₀ + b₁₁)

c₀₀ = m₁ + m₄ - m₅ + m₇
c₀₁ = m₃ + m₅
c₁₀ = m₂ + m₄
c₁₁ = m₁ + m₃ - m₂ + m₆

Complexity:

M(n) = nlog₂7 ≈ n2.807 vs. Θ(n³) of brute-force

💡 Intuitive Analogy: Like using a clever trick to multiply two numbers with fewer multiplications — instead of 8, you do 7, which makes a big difference for large matrices.

Master Theorem

For recurrences of form: T(n) = aT(n/b) + f(n), where a ≥ 1, b > 1

Compare f(n) with nlogba:

CaseConditionSolution
1f(n) = O(nlogba - ε)T(n) = Θ(nlogba)
2f(n) = Θ(nlogba lgkn)T(n) = Θ(nlogba lgk+1n)
3f(n) = Ω(nlogba + ε)
and af(n/b) ≤ cf(n) for c < 1		T(n) = Θ(f(n))

Examples:

  • T(n) = 4T(n/2) + n ⇒ a=4,b=2 ⇒ nlog₂4 = n² ⇒ Case 1: T(n) = Θ(n²)
  • T(n) = 4T(n/2) + n² ⇒ Case 2: T(n) = Θ(n² log n)
  • T(n) = 4T(n/2) + n³ ⇒ Case 3: T(n) = Θ(n³)
  • T(n) = 2T(n/2) + n ⇒ a=2,b=2 ⇒ nlog₂2 = n, f(n)=n ⇒ Case 2: T(n) = Θ(n log n) (Merge Sort)
💡 Intuitive Analogy: The Master Theorem is like a recipe that tells you how to solve recurrence "soup" — just compare the "ingredient" f(n) with the "base" nlogba and pick the right case.

Recurrence Relations: Substitution Method

Method: Guess the form of solution, verify by induction, solve for constants.

Example: T(n) = 4T(n/2) + n

  1. Guess O(n³): Assume T(k) ≤ ck³ for k < n
  2. T(n) = 4c(n/2)³ + n = (c/2)n³ + n = cn³ - ((c/2)n³ - n) ≤ cn³ (if c ≥ 2)
  3. Tighter: Prove O(n²) — strengthen hypothesis: T(k) ≤ c₁k² - c₂k

Recursion Tree Method:

Solve T(n) = T(n/4) + T(n/2) + n²:

T(n) = n² + (5/16)n² + (25/256)n² + ... = n²(1 + 5/16 + (5/16)² + ...) = Θ(n²)

Closest Pair Problem (D&C)

Algorithm:

  1. Sort points by x-coordinate
  2. Divide into two halves by vertical line
  3. Recursively find closest pair in each half
  4. d = min(d₁, d₂); Check strip of width 2d across dividing line
  5. For each point in strip, check at most 6 points on other side

Complexity:

T(n) = 2T(n/2) + Θ(n) ⇒ T(n) = Θ(n log n)

Quickhull Algorithm

Convex Hull: Smallest convex set containing given points.

Algorithm:

  1. Sort points by x-coordinate
  2. Identify extreme points P₁ (leftmost) and P₂ (rightmost)
  3. Compute upper hull: find Pmax farthest from line P₁P₂
  4. Recursively compute hull of points left of P₁Pmax and PmaxP₂
  5. Compute lower hull similarly

Complexity:

  • Worst-case: Θ(n²)
  • Average-case: Θ(n)

AVL Trees

Definition: A binary search tree where balance factor of every node (height of left subtree - height of right subtree) is -1, 0, or +1.

Rotations:

  • R-rotation (single right): After insertion into left subtree of left child (balance = +2)
  • L-rotation (single left): After insertion into right subtree of right child (balance = -2)
  • LR-rotation (double): After insertion into right subtree of left child
  • RL-rotation (double): After insertion into left subtree of right child

Complexity:

Search, Insert, Delete: Θ(log n)

2-3 Trees

Definition: Tree with two kinds of nodes:

  • 2-node: Contains single key K, has two children (left < K < right)
  • 3-node: Contains two ordered keys K₁ < K₂, has three children

Property: All leaves are on the same level — perfectly height-balanced.

Sorting in Linear Time

Bucket Sort (BinSort):

If all keys are 1…K, place each key in its bin, then concatenate bins.

Time: O(n+K) — linear if K = O(n)

Stable: Yes

Radix Sort:

Sort numbers digit by digit from least significant to most significant using a stable sort (like counting sort) for each digit.

Time: O(d·(n+K)) where d = number of digits

Counting Sort:

Count occurrences of each key value, compute prefix sums, place elements in correct positions.

Time: O(n+K) — linear if K = O(n)

Stable: Yes

Unit 4: Dynamic Programming & Greedy Techniques (Click to expand)

Dynamic Programming Overview

Definition: A general algorithm design technique for solving problems defined by recurrences with overlapping subinstances.

  • Invented by Richard Bellman in the 1950s
  • "Programming" means "planning"

Main Idea:

  1. Set up recurrence relating larger instance to smaller instances
  2. Solve smaller instances once
  3. Record solutions in a table
  4. Extract solution to initial instance from table

Examples:

  • Fibonacci numbers, Binomial coefficient
  • Knapsack problem, LCS
  • Warshall's algorithm (transitive closure)
  • Floyd's algorithm (all-pairs shortest paths)
  • Optimal BST
💡 Intuitive Analogy: Like solving a big puzzle by first solving all the small overlapping pieces once, storing their answers, and then using them to build the final solution — avoiding repeated work.

DP: Knapsack Problem

Problem: Given n items with weights w₁...wₙ and values v₁...vₙ, and capacity W, find most valuable subset.

Recurrence:

V[i,j] = max{V[i-1,j], vᵢ + V[i-1,j-wᵢ]}  if j-wᵢ ≥ 0
         V[i-1,j]                           if j-wᵢ < 0
V[0,j] = 0, V[i,0] = 0

Example: W=5

ItemWeightValue
12$12

Optimal value: $37 (items 1,2,4)

Complexity:

Time & Space: O(nW) — pseudopolynomial

💡 Intuitive Analogy: Like packing a suitcase with the most valuable items, but you can only take each item once — DP tries all combinations efficiently.
// 0/1 Knapsack DP (bottom-up)
int knapSack(int W, int wt[], int val[], int n) {
    int dp[n+1][W+1];
    for (int i=0; i<=n; i++) {
        for (int w=0; w<=W; w++) {
            if (i==0 || w==0) dp[i][w] = 0;
            else if (wt[i-1] <= w)
                dp[i][w] = max(val[i-1] + dp[i-1][w-wt[i-1]], dp[i-1][w]);
            else dp[i][w] = dp[i-1][w];
        }
    }
    return dp[n][W];
}

DP: Longest Common Subsequence (LCS)

Problem: Find maximum length common subsequence between two sequences.

Recurrence:

len(i,j) = 0                                    if i=0 or j=0
         = len(i-1,j-1) + 1                     if aᵢ = bⱼ
         = max(len(i,j-1), len(i-1,j))          if aᵢ ≠ bⱼ

Example:

String 1: "president", String 2: "providence"
LCS: "priden"

Complexity:

Time & Space: O(mn)

💡 Intuitive Analogy: Like finding the longest overlapping sequence in two strings — DP builds a table of matches.
// LCS length (bottom-up)
int lcs(const string& X, const string& Y) {
    int m = X.size(), n = Y.size();
    int dp[m+1][n+1];
    for (int i=0; i<=m; i++) {
        for (int j=0; j<=n; j++) {
            if (i==0 || j==0) dp[i][j] = 0;
            else if (X[i-1] == Y[j-1]) dp[i][j] = 1 + dp[i-1][j-1];
            else dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
        }
    }
    return dp[m][n];
}

Warshall's Algorithm (Transitive Closure)

Problem: Find transitive closure of a relation / existence of all paths in a digraph.

Recurrence:

R⁽ᵏ⁾[i,j] = R⁽ᵏ⁻¹⁾[i,j] OR (R⁽ᵏ⁻¹⁾[i,k] AND R⁽ᵏ⁻¹⁾[k,j])

R⁽⁰⁾ = adjacency matrix A, R⁽ⁿ⁾ = transitive closure T

Complexity:

Time: Θ(n³), Space: Θ(n²)

Floyd's Algorithm (All-Pairs Shortest Paths)

Problem: Find shortest paths between every pair of vertices in weighted graph.

Recurrence:

D⁽ᵏ⁾[i,j] = min(D⁽ᵏ⁻¹⁾[i,j], D⁽ᵏ⁻¹⁾[i,k] + D⁽ᵏ⁻¹⁾[k,j])

Complexity:

Time: Θ(n³), Space: Θ(n²)

Note: Works with negative edges but not negative cycles.

💡 Intuitive Analogy: Like trying to find the shortest path between every pair of cities by considering each city as a potential intermediate stop — updating distances if going through that city is shorter.
// Floyd-Warshall (all-pairs shortest path)
const int INF = 1e9;
void floydWarshall(int dist[][V]) {
    for (int k=0; k<V; k++)
        for (int i=0; i<V; i++)
            for (int j=0; j<V; j++)
                if (dist[i][k] + dist[k][j] < dist[i][j])
                    dist[i][j] = dist[i][k] + dist[k][j];
}

Optimal Binary Search Tree

Problem: Given n keys a₁ < … < aₙ and probabilities p₁,…,pₙ, find BST with minimum average comparisons.

Recurrence:

C[i,j] = min {C[i,k-1] + C[k+1,j]} + Σpₛ  for 1 ≤ i ≤ j ≤ n
C[i,i] = pᵢ

Complexity:

Time: Θ(n³) (can be reduced to Θ(n²)), Space: Θ(n²)

Greedy Technique Overview

Definition: Constructs solution piece by piece through locally optimal (greedy), feasible, and irrevocable choices.

Optimal Solutions:

  • Change making (normal denominations)
  • Minimum Spanning Tree (Prim's, Kruskal's)
  • Single-source shortest paths (Dijkstra's)
  • Huffman codes

Approximations:

  • TSP, Knapsack, other combinatorial optimization
💡 Intuitive Analogy: Like making decisions that seem best right now, without worrying about the future — it works for some problems (like finding the shortest route with Dijkstra) but not for others (like knapsack).

Prim's MST Algorithm

Algorithm:

  1. Start with tree T₁ consisting of one vertex
  2. Grow tree one vertex at a time
  3. On each iteration, add vertex not in tree that is closest to those already in tree

Complexity:

  • O(n²) with weight matrix and array PQ
  • O(m log n) with adjacency lists and min-heap PQ
💡 Intuitive Analogy: Like building a tree by always connecting the nearest unconnected node — it's like growing a tree from a seed.

Kruskal's MST Algorithm

Algorithm:

  1. Sort edges in nondecreasing order of weight
  2. Add next edge unless it creates a cycle (skip if it does)

Complexity:

O(m log m) — dominated by sorting

Cycle checking: Union-Find data structure

💡 Intuitive Analogy: Like adding the cheapest roads one by one, but only if they don't create a loop — eventually you connect all cities with minimum total cost.

Dijkstra's Algorithm (Shortest Paths)

Problem: Single Source Shortest Paths in weighted graph.

Algorithm: Similar to Prim's but different labeling: find vertex u with smallest dv + w(v,u) where v is already in tree.

Complexity:

  • O(|V|²) with weight matrix
  • O(|E| log |V|) with adjacency lists and min-heap

Note: Does NOT work with negative weights (use Floyd's instead)

💡 Intuitive Analogy: Like finding the shortest route on a map by always expanding the closest unvisited node — it's like following the most promising path.

Huffman Codes

Problem: Find optimal prefix-free binary code for given character frequencies.

Algorithm:

  1. Initialize n one-node trees with characters and frequencies
  2. Repeat n-1 times: join two smallest-weight trees, sum their weights
  3. Label left edges 0, right edges 1

Example: A(0.35), B(0.1), C(0.2), D(0.2), _(0.15)

Codes: A=11, B=100, C=00, D=01, _=101
Average bits: 2.25 (vs 3 for fixed-length, 25% compression)

💡 Intuitive Analogy: Like giving shorter codes to frequent letters (like 'e') and longer codes to rare ones — this saves space, just like Morse code.

Unit 5: Backtracking & Branch & Bound (Click to expand)

Backtracking

Definition: Technique for solving problems with a large search space by systematically trying and eliminating possibilities.

Strategy:

  1. Try each choice at a junction (decision point)
  2. If stuck, "backtrack" to junction and try next choice
  3. If all choices tried and no solution, there is none

Analogy: Maze solving — try a path, backtrack when stuck, try another.

💡 Intuitive Analogy: Like exploring a maze: when you hit a dead end, you retrace your steps and try a different fork.

8 Queens Problem

Problem: Place 8 queens on chess board such that no two attack each other.

Constraints: No two queens share same row, column, or diagonal.

Backtracking Solution:

void solveItRec(int perm[], int location, struct onesquare usedList[]) {
    if (location == SIZE) { printSol(perm); return; }
    for (int i = 0; i < SIZE; i++) {
        if (usedList[i] == false) {
            if (!conflict(perm, location, i)) {
                perm[location] = i;
                usedList[i] = true;
                solveItRec(perm, location+1, usedList);
                usedList[i] = false;
            }
        }
    }
}

Analysis:

Brute force: 8! = 40,320 permutations
Backtracking: ~15,720 placements examined
With pruning: ~5,508 placements

💡 Intuitive Analogy: Like placing queens one by one, checking conflicts, and if a conflict arises, moving the last queen to the next possible column — undo and try again.
// Check if queen at (row, col) is safe
bool isSafe(int board[], int row, int col) {
    for (int i=0; i<row; i++) {
        if (board[i] == col || abs(board[i]-col) == abs(i-row))
            return false;
    }
    return true;
}
void solveNQueens(int row, int board[], int N) {
    if (row == N) { // found a solution return; }
    for (int col=0; col<N; col++) {
        if (isSafe(board, row, col)) {
            board[row] = col;
            solveNQueens(row+1, board, N);
            // backtrack: board[row] = -1; (implicitly overwritten)
        }
    }
}

Branch and Bound

Definition: An algorithmic technique for solving optimization problems by systematically enumerating candidate solutions and pruning branches that cannot yield a better solution than the current best.

Key concepts:

  • Bounding function: Estimates lower bound (for minimization) or upper bound (for maximization)
  • Pruning: Discard branches whose bound is worse than current best
  • Used for: TSP, Knapsack, Assignment Problem
💡 Intuitive Analogy: Like pruning a tree: if a branch can't possibly beat the best solution you've already found, you cut it off and save time.

Vertex Cover Problem

Definition: A vertex cover of a graph G = (V,E) is a subset V' ⊆ V such that for every edge (u,v) ∈ E, at least one of u or v is in V'.

Decision problem: Does there exist a vertex cover of size k?

Complexity: NP-Complete

Bin Packing Problem

Definition: Objects of different volumes must be packed into a finite number of bins of volume V, minimizing number of bins used.

Variations: 2D packing, linear packing, packing by weight

Heuristics: First-Fit Decreasing — sort items by decreasing size, then place each in first bin it fits.

Complexity: NP-Hard; First-Fit: Θ(n log n)

Unit 6: Advanced Topics (Click to expand)

Euclidean Algorithm (Detailed)

Algorithm for GCD:

integer euclid(pos. integer m, pos. integer n)
    x = m, y = n
    while (y > 0)
        r = x mod y
        x = y
        y = r
    return x

Dry Run: gcd(244, 117)

Step 1: x=244, y=117, r=244 mod 117 = 10
Step 2: x=117, y=10,  r=117 mod 10 = 7
Step 3: x=10,  y=7,   r=10 mod 7 = 3
Step 4: x=7,   y=3,   r=7 mod 3 = 1
Step 5: x=3,   y=1,   r=3 mod 1 = 0
Return: 1  (244 and 117 are relatively prime)

Correctness:

gcd(x',y') = gcd(y, x mod y) = gcd(y, x - qy) = gcd(y, x) = gcd(x,y)

Modular Arithmetic

Definition: System of arithmetic for integers where numbers "wrap around" upon reaching the modulus.

a ≡ b (mod n) iff n divides (a − b)

Properties:

  • If a ≡ b (mod n) and c ≡ d (mod n), then:
  • a + c ≡ b + d (mod n)
  • a − c ≡ b − d (mod n)
  • a × c ≡ b × d (mod n)

Chinese Remainder Theorem

Statement: Suppose n₁,n₂,…,nk are pairwise coprime positive integers. For any integers a₁,a₂,…,ak, there exists integer x solving:

x ≡ a₁ (mod n₁)
x ≡ a₂ (mod n₂)
...
x ≡ ak (mod nk)

Example:

Find n such that: n ≡ 2 (mod 3), n ≡ 3 (mod 5), n ≡ 2 (mod 7)
Answer: n = 23

Complexity Classes (P, NP, NPC)

Three Classes:

  • P: Problems solvable in polynomial time
  • NP: Problems verifiable in polynomial time
  • NPC (NP-Complete): Problems in NP and as hard as any problem in NP

Key Relationships:

  • P ⊆ NP (sure)
  • NPC ⊆ NP (sure)
  • P = NP? — Millennium Prize Problem (likely P ≠ NP)

NP-Complete Problems Include:

  • Circuit-SAT (first proven NPC)
  • SAT (Formula Satisfiability)
  • 3-CNF-SAT
  • CLIQUE
  • Vertex Cover
  • Hamiltonian Cycle
  • TSP (Traveling Salesman Problem)
  • Subset Sum
  • Bin Packing

Implication of NPC:

If a problem is NP-Complete, don't waste time trying to find an efficient (polynomial) algorithm. Instead, focus on approximate or heuristic solutions.

💡 Intuitive Analogy: P is like solving a puzzle quickly, NP is like checking a solution quickly. NPC problems are the hardest puzzles — if you can solve one quickly, you can solve them all (but probably you can't).

NP-Completeness Proofs

Steps to Prove a Problem X is NPC:

  1. Prove X ∈ NP (certificate can be verified in poly time)
  2. Prove X is NP-hard: Select known NPC problem P', construct polynomial reduction from P' to X

Reduction:

A reduction from A to B transforms instance α of A to instance β of B such that:

  • Transformation takes polynomial time
  • α has YES answer iff β has YES answer

Reduction Chain:

Circuit-SAT → SAT → 3-CNF-SAT → CLIQUE → Vertex Cover → Hamiltonian Cycle → TSP → Subset Sum

📝 One-Line Revision Notes (30 Key Points)

  1. Algorithm: Sequence of finite, unambiguous instructions for solving a problem.
  2. Big O: Upper bound on growth rate — f(n) ≤ c·g(n).
  3. Big Ω: Lower bound — f(n) ≥ c·g(n).
  4. Big Θ: Tight bound — both O and Ω.
  5. Selection Sort: O(n²), stable, in-place — find min and swap.
  6. Insertion Sort: O(n²), stable, in-place — insert into sorted portion.
  7. Merge Sort: Θ(n log n), stable, O(n) space — divide, sort, merge.
  8. Quick Sort: Θ(n log n) avg, O(n²) worst — partition around pivot.
  9. Binary Search: O(log n) — requires sorted array.
  10. Master Theorem: T(n) = aT(n/b) + f(n) — compare f(n) with nlogba.
  11. Strassen's MM: O(n^2.807) — 7 multiplications instead of 8.
  12. KMP: O(m+n) — uses prefix function to avoid backtracking on text.
  13. Rabin-Karp: O(n+m) avg — hashing-based string matching.
  14. DP Knapsack: O(nW) — fill table bottom-up.
  15. LCS: O(mn) — use recurrence based on character match.
  16. Floyd-Warshall: O(n³) — all-pairs shortest paths.
  17. Warshall's: O(n³) — transitive closure.
  18. Prim's MST: Grow tree by adding closest vertex.
  19. Kruskal's MST: Sort edges, add non-cycle edges.
  20. Dijkstra's: O((V+E)log V) — single source shortest paths.
  21. Huffman: Greedy — merge smallest frequency nodes.
  22. Backtracking: Try, if fail backtrack and try next.
  23. AVL Tree: BST with balance -1,0,1 — rotations fix imbalance.
  24. Radix Sort: Sort digit by digit, O(d(n+K)).
  25. Bucket Sort: O(n+K) — distribute into K buckets.
  26. P vs NP: Open problem — can verification = solution?
  27. NP-Complete: Hardest NP problems — no poly solution known.
  28. Chinese Remainder Theorem: Solves system of congruences with coprime moduli.
  29. Euclid's GCD: O(log n) — gcd(m,n) = gcd(n, m mod n).
  30. Modular Arithmetic: Numbers wrap at modulus — a ≡ b (mod n).
