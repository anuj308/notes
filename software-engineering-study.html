🔄 Life Cycle Models

Model Key Idea Advantages Disadvantages
Classical Waterfall Sequential phases: feasibility → req. analysis → design → coding → testing → maintenance Simple, easy to understand, each phase has clear deliverables No feedback paths, difficult to accommodate changes, no overlapping, limited customer interaction
Iterative Waterfall Waterfall with feedback paths between phases Errors can be corrected by going back, phase containment of errors, most widely used Suitable only for well-understood problems
Prototyping Build a working prototype first, refine based on customer feedback Helps clarify unclear requirements, reduces overall cost for unclear systems Additional cost of building prototype, may lead to poor design if prototype is kept
Evolutionary Break system into modules, incrementally implement and deliver Users get early access, core modules tested thoroughly, incremental delivery Difficult to subdivide problems into functional units, useful only for large problems
Spiral Iterative loops with risk assessment in each quadrant Subsumes all models, handles risks explicitly, uses prototyping for risk reduction Complex, requires risk assessment expertise, can be expensive

🔗 Cohesion Types (Best → Worst)

TypeDescriptionExample
Functional ✅All elements cooperate to achieve a single taskManaging-book-lending module with issue-book, return-book, query-book
SequentialOutput of one element is input to the nextSort → Search → Display
CommunicationalAll functions refer to/update the same data structureFunctions defined on an array or stack
ProceduralFunctions are part of a procedure (algorithm) in a certain orderOrder processing: login → check-order → print-bill → update-inventory
TemporalFunctions must be executed at the same timeBooting process, start-up/shut-down of system
LogicalElements perform similar operationsModule containing print functions for grade sheet, salary slip, annual reports
Coincidental ❌Random collection of unrelated functionsRandom-operations: Issue-book, create member, request librarian leave

TypeDescription
