📄 Data Communications

Definition: Data communications is the exchange of data between two devices via some form of transmission medium (wire cable or wireless).

Four Fundamental Characteristics

  • Delivery: Data must be delivered to the correct destination
  • Accuracy: Data must be delivered accurately (no corruption)
  • Timeliness: Data must be delivered in a timely manner (real-time)
  • Jitter: Variation in packet arrival time must be controlled

Five Components of a Data Communication System

  • Message: Text, Number, Image, Audio, Video
  • Sender
  • Receiver
  • Transmission Medium
  • Protocol - Set of rules governing communication

Data Flow Modes

Half-DuplexBoth directions but one at a timeWalkie-talkie, Two-way radio
Full-DuplexBoth directions simultaneouslyTelephone, Video call

🌐 Network Definition

A network is a set of devices (often called nodes) connected by communication links. A node can be a computer, printer, or any device capable of sending/receiving data.

Network Criteria

  • Performance: Measured by delay and throughput. Depends on transmit time, response time, number of users, type of medium
  • Reliability: Failure rate, time to recover, measured by availability/robustness
  • Security: Data protection against corruption, errors, unauthorized access

🔗 Protocols

Definition: A protocol is a set of rules that govern data communications. It determines what is communicated, how it is communicated, and when it is communicated.

Three Key Elements of a Protocol

  • Syntax: Structure or format of data - how to read the bits (field delineation)
  • Semantics: Interprets the meaning of bits - which fields define what action
  • Timing: When data should be sent and at what speed

OSI Model Overview

ISO (International Standards Organization) established the OSI (Open Systems Interconnection) model in the late 1970s. It is a standard that covers all aspects of network communications.

ISO is the organization; OSI is the model.

Seven Layers of the OSI Model

LayerNameFunctionData Unit
7ApplicationProvides services to the user (NVT, File Transfer, Mail, Directory Services)Message
6PresentationTranslation, Encryption, CompressionMessage
5SessionDialog control, Synchronization (checkpoints)Message
4TransportService-point addressing, Segmentation/reassembly, Connection/Flow/Error controlSegment
3NetworkLogical addressing, RoutingPacket/Datagram

Layer-by-Layer Explanation

1. Physical Layer

Responsible for movement of individual bits from one hop (node) to the next.

  • Type of transmission media
  • Representation of bits (encoding)
  • Data rate (bits per second)
  • Synchronization of bits (sender/receiver clock)
  • Line configuration (point-to-point / multipoint)
  • Topology
  • Transmission mode (simplex, half-duplex, full-duplex)

2. Data Link Layer

Responsible for moving frames from one hop (node) to the next. Hop-to-hop delivery.

  • Framing: Dividing bit stream into frames
  • Physical addressing: MAC addresses
  • Flow control: Preventing sender from overwhelming receiver
  • Error control: Detection and retransmission of damaged/lost frames
  • Access control: Managing which device transmits when

3. Network Layer

Responsible for delivery of individual packets from source host to destination host. Source-to-destination delivery.

  • Logical addressing: IP addresses
  • Routing: Finding the best path

4. Transport Layer

Responsible for delivery of a message from one process to another (process-to-process).

  • Service-point addressing (port numbers)
  • Segmentation and reassembly
  • Connection control
  • Flow control
  • Error control

5. Session Layer

Responsible for dialog control and synchronization.

  • Dialog control: Decides whose turn to transmit
  • Synchronization: Introducing checkpoints to recover from failures

6. Presentation Layer

Responsible for translation, compression, and encryption.

  • Translation: Converting between different data formats
  • Encryption: Securing data
  • Compression: Reducing data size

7. Application Layer

Responsible for providing services to the user.

  • Network Virtual Terminal (Telnet)
  • File transfer, access, and management (FTP)
  • Mail services (SMTP)
  • Directory Services (DNS)

OSI Model - Data Encapsulation

When data moves from the application layer down to the physical layer, each layer adds its own header (and sometimes trailer). This process is called encapsulation.

📜
Data Flow in OSI Model:
Application ↓ (Data)
Presentation ↓ (Data)
Session ↓ (Data)
Transport ↓ (Segment = header + data)
Network ↓ (Packet = header + segment)
Data Link ↓ (Frame = header + packet + trailer)
Physical ↓ (Bits)
Each layer adds its own header during transmission (encapsulation). Headers are stripped at the receiving end (decapsulation).

TCP/IP Model

The TCP/IP protocol suite was originally defined as having four layers: host-to-network, internet, transport, and application. When compared to OSI, it can be seen as five layers: physical, data link, network, transport, and application.

Addressing in TCP/IP

Four levels of addresses are used:

  1. Physical Address (MAC): Data Link Layer - 48-bit unique hardware address
  2. Logical Address (IP): Network Layer - IPv4 (32-bit) or IPv6 (128-bit)
  3. Port Address: Transport Layer - 16-bit (0-65535)
  4. Specific Address: Application Layer - e.g., email address, URL

📚 Unit 2: Physical Layer

Physical Layer: Signal & Media

📚 Data & Signals

Analog and Digital Data

Data can be analog (continuous values) or digital (discrete states).

  • Analog data: Continuous - e.g., voice, video
  • Digital data: Discrete states - e.g., text, integers
  • Analog signals: Infinite number of values in a range
  • Digital signals: Limited number of values

Periodic Analog Signals

A sine wave is represented by: Peak Amplitude, Frequency, Phase, Wavelength

  • Peak Amplitude: Absolute value of highest intensity (measured in volts)
  • Period (T): Amount of time to complete one cycle (seconds)
  • Frequency (f): Number of periods in 1 second (Hertz). f = 1/T
  • Phase: Position of the waveform relative to time 0 (degrees or radians)
  • Wavelength (λ): Distance signal travels in one period. λ = propagation speed × period

Key Formula

Frequency & Period: f = 1/T, T = 1/f

Wavelength: λ = c / f (where c = speed of light in the medium)

Bandwidth: Difference between highest and lowest frequencies in a composite signal

Fourier Analysis

  • Any composite signal is a combination of simple sine waves with different frequencies, amplitudes, and phases
  • If the composite signal is periodic → decomposition gives discrete frequencies
  • If the composite signal is nonperiodic → decomposition gives continuous frequencies

Digital Signals

A digital signal can have more than two levels. Each signal level represents multiple bits.

Bit Rate: Number of bits sent in 1 second (bps). Most digital signals are non-periodic.

Bit Length: Distance one bit occupies on transmission medium = propagation speed × bit duration

Baseband vs Broadband Transmission

  • Baseband: Digital signal sent directly over a low-pass channel. Bandwidth required = bit rate/2
  • Broadband: Digital signal modulated onto an analog carrier for bandpass channels. Requires modulation (modem)

Data Rate Limits

Nyquist Bit Rate (Noiseless Channel)

Bit Rate = 2 × Bandwidth × log₂(L) where L = number of signal levels

Shannon Capacity (Noisy Channel)

Capacity = Bandwidth × log₂(1 + SNR) where SNR = Signal-to-Noise Ratio

SNR in dB

SNRₔₘ = 10 log₁₀(SNR)

Examples

Nyquist: Channel with 3000 Hz bandwidth, 2 signal levels → Max bit rate = 2 × 3000 × log₂2 = 6000 bps

Shannon: Telephone line with 3000 Hz bandwidth, SNR = 3162 → Capacity = 3000 × log₂(1+3162) ≈ 34,860 bps

Digital-to-Digital Conversion

Line Coding Schemes

CategorySchemeDescription
UnipolarNRZ (Non-Return to Zero)All signal levels on one side of time axis. Prone to DC components, no synchronization
PolarNRZ-L+V for 1, -V for 0. Voltage level determines bit value. DC component problem
PolarNRZ-IInversion = 1, no inversion = 0. Better than NRZ-L

Key Point

Manchester and Differential Manchester have a transition at the middle of each bit used for synchronization. Minimum bandwidth is 2× that of NRZ. No DC component.

Analog-to-Digital Conversion: PCM

Pulse Code Modulation (PCM) steps: Sampling → Quantization → Binary Encoding

  • Sampling: Nyquist theorem - sampling rate must be at least 2x highest frequency
  • Quantization: Mapping infinite amplitude values to finite set. Δ = (max - min)/L
  • Encoding: Each quantized value represented in binary

📚 Transmission Impairments & Multiplexing

Transmission Impairments

Signal transmitted through imperfect media causes impairment: what is sent is not what is received.

Types of Impairment

ImpairmentDescription
AttenuationLoss of energy = weaker signal. Compensated by amplifiers. Measured in decibels (dB).
dB = 10 log₁₀(P₂/P₁)
DistortionSignal changes shape. Each frequency component has different propagation speed → different delays
NoiseThermal noise, crosstalk, impulse noise (spikes from power lines, lightning), induced noise

Signal-to-Noise Ratio (SNR)

SNR = Signal Power / Noise Power. SNRₔₘ = 10 log₁₀(SNR). Higher SNR = better signal quality.

Decibel Examples

  • Power reduced to half: dB = 10 log₁₀(0.5) = -3 dB (3 dB loss)
  • Power increased 10×: dB = 10 log₁₀(10) = 10 dB (10 dB gain)
  • dBm: dBm = 10 log₁₀(Pm) where Pm is power in milliwatts

Multiplexing

Definition: Set of techniques that allows simultaneous transmission of multiple signals across a single data link.

📚 Framing & Data Link Control

Data Link Layer Design Issues

  • Services provided to the Network Layer
  • Framing: Packing bits into frames
  • Error Control
  • Flow Control

Framing

The data link layer needs to pack bits into frames so each frame is distinguishable.

Types of Framing

TypeDescriptionTechnique
Fixed-Size FramingAll frames same sizeNo boundary markers needed
Variable-Size Framing (Character-Oriented)Frames of varying size, uses flags and byte stuffingByte stuffing: Add extra byte when flag or escape character appears in text
Variable-Size Framing (Bit-Oriented)Frames of varying size, uses flag 01111110Bit stuffing: Add extra 0 after five consecutive 1s

Bit Stuffing Rule

When data contains pattern 01111110 (same as flag), add an extra 0 after five consecutive 1s: 011111 → 0111110. Receiver removes the extra 0.

📚 Error Detection & Correction

Exam Important

Types of Errors

  • Single-bit error: One bit changed. Rare in serial transmission (noise must be very short)
  • Burst error: Two or more bits changed. Measured from first corrupted to last corrupted bit. Most common in serial transmission

Error Detection Techniques

TechniqueDescriptionPerformance
VRC (Vertical Redundancy Check)Parity bit added to each data unit; makes total number of 1s even (or odd)Detects burst errors only if odd number of errors
LRC (Longitudinal Redundancy Check)Additional parity bits for each bit position across all data unitsHigher likelihood of detecting burst errors. Fails if two bits in same position in different data units are damaged
CRC (Cyclic Redundancy Check)Generator creates n-bit FCS so (k+n) bits are divisible by predetermined number. Receiver divides by same numberVery powerful; detects most common errors
ChecksumData divided into k sections of n bits; all sections added (1's complement); sum complemented = checksumDetects all odd-numbered errors; most even-numbered errors

Error Correction (Hamming Code)

For m data bits and r redundancy bits: 2⁶ ≥ m + r + 1

Hamming Code can detect and correct single-bit errors.

📚 Flow Control Protocols

Flow & Error Control

Flow control: Restricts amount of data sender can send before waiting for acknowledgment.
Error control: Based on Automatic Repeat Request (ARQ) - retransmission of data.

Protocol Taxonomy

Channel TypeProtocolDescription
NoiselessSimplest ProtocolNo flow or error control
Stop-and-WaitSender sends one frame, waits for confirmation
NoisyStop-and-Wait ARQUses sequence numbers (0,1,0,1...). Acknowledgment number is opposite of sequence number
Go-Back-N ARQSends multiple frames before receiving acknowledgment. Uses sliding window. Send window size < 2⁶. Receiver window size = 1. On error, retransmits from that point
Selective Repeat ARQOnly retransmits damaged frames. Both sender and receiver windows must be ≤ 2⁶⁻¹

Pipelining

Task begins before end of first task. Stop-and-Wait doesn't use pipelining; Go-Back-N and Selective Repeat do. This improves efficiency.

Bandwidth-Delay Product

Bandwidth-Delay Product = Bandwidth × Round-Trip Time. Measures number of bits that can be sent while waiting for acknowledgment.

Utilization = (Frame Size) / (Bandwidth-Delay Product) if using Stop-and-Wait

📚 Ethernet & MAC Sublayer

Standard Ethernet (IEEE 802.3)

Original Ethernet created in 1976 at Xerox PARC. Four generations.

802.3 MAC Frame Format

  • Preamble: 7 bytes (10101010 pattern)
  • SFD: Start Frame Delimiter (1 byte = 10101011)
  • Destination Address: 6 bytes (MAC)
  • Source Address: 6 bytes (MAC)
  • Length/Type: 2 bytes
  • Data: 46-1500 bytes
  • CRC: 4 bytes

Frame Size Limits

MAC Address Types

  • Unicast: LSB of first byte = 0 (for a single destination)
  • Multicast: LSB of first byte = 1 (for a group)
  • Broadcast: All bits 1 (special case of multicast)

Address Notation

Example: A0:B1:C2:D3:E4:F5 (6 bytes = 48 bits in hexadecimal)

📚 Subnetting

Exam ImportantPractice Problems

Subnetting Problems

Subnetting is applicable for both Classful and Classless addressing.

Example 1

Q: What is the Network ID, Broadcast Address, First Usable IP, Last Usable IP for node 192.168.1.15/26?

Solution:

  • /26 = 255.255.255.192 (128+64 = 192)
  • Block size = 64
  • Network ID = 192.168.1.0
  • Broadcast = 192.168.1.63
  • First Usable = 192.168.1.1
  • Last Usable = 192.168.1.62

Example 2

Q: How many subnets and hosts from 192.168.1.0 with mask 255.255.255.224?

Solution:

  • Class C default = /24 (255.255.255.0)
  • 224 = 128+64+32 → 3 subnet bits
  • Host bits = 8 - 3 = 5
  • Subnets = 2³ = 8
  • Hosts per subnet = 2⁭ - 2 = 30

Example 3 - Large Network

Q: 172.16.0.0 network needs 900 subnets with at least 50 hosts/subnet.

Solution:

  • Class B default = /16
  • Need 2⁶ ≥ 900 → 10 subnet bits (2¹⁰ = 1024)
  • Mask: /16 + 10 = /26 = 255.255.255.192

📚 Distance Vector Routing & RIP

Distance Vector Routing

  • Each node maintains a vector/table of minimum distances to every node
  • Next-hop routing: table guides packets to next stop
  • Each node shares its table with immediate neighbors periodically (every 30s) and on change (triggered update)
  • Limitation: Count-to-infinity problem (two-node and three-node instability)

Solutions to Instability

  • Define Infinity: Set max hop count to a smaller number (e.g., 16 for RIP)
  • Split Horizon: Don't advertise a route back to the node it came from
  • Poison Reverse: Advertise the route with distance = infinity if source was that node

RIP (Routing Information Protocol)

  • Intradomain protocol based on distance vector
  • Metric = hop count (number of links to destination)
  • Infinity = 16 (max 15 hops)
  • Simple protocol, suitable for small to medium networks

📚 Link State Routing & OSPF

Link State Routing

  • Each node has entire topology of the domain
  • Four steps:
    1. Create Link State Packet (LSP): node identity, list of links, sequence number, age
    2. Flood LSPs to every router efficiently and reliably
    3. Form shortest path tree (using Dijkstra algorithm)
    4. Calculate routing table from shortest path tree
  • LSPs generated on topology change and periodically

OSPF (Open Shortest Path First)

  • Intradomain routing protocol based on link state
  • Allows administrator to assign cost (metric) based on type of service
  • Router can have multiple routing tables for different TOS
  • Supports hierarchy through areas within autonomous system
  • Link types: Point-to-point, Transient, Stub, Virtual

📚 Path Vector Routing & BGP

Path Vector Routing

Problem in Distance Vector: Node instability.
Problem in Link State: Heavy traffic due to flooding, extra resources needed.

Solution: Path Vector - Similar to distance vector but includes the complete path (list of ASs) in routing updates. This prevents loops.

  • One speaker node per AS creates/ advertises routing table to neighboring ASs
  • Steps: Initialization → Sharing → Updating (loop prevention, policy routing, optimum path)

BGP (Border Gateway Protocol)

  • Interdomain routing protocol using path vector routing
  • First appeared in 1989; now on version 4
  • Path Attributes: Well-known (mandatory, discretionary) and Optional (transitive, non-transitive)
  • BGP Sessions: Use TCP connections (reliable). Called semi-permanent connections
  • Types: Internal BGP (iBGP) and External BGP (eBGP)

TCP Connection Establishment (Three-Way Handshake)

  1. Client → Server: SYN (seq = x) - Client requests connection
  2. Server → Client: SYN + ACK (seq = y, ack = x+1) - Server acknowledges and synchronizes
  3. Client → Server: ACK (seq = x+1, ack = y+1) - Connection established

SYN consumes one sequence number. ACK with no data consumes none.

TCP Connection Termination (Three-Way Handshake)

  1. Client → Server: FIN
  2. Server → Client: ACK
  3. Server → Client: FIN
  4. Client → Server: ACK

FIN consumes one sequence number. Half-close is possible when only one side terminates.

Example

TCP header in hex: 05320017 00000001 00000000 500207FF 00000000

  • Dest port: 0x0017 = 23 (Telnet)
  • Seq number: 1
  • Ack number: 0
  • HLEN: 5 (20 bytes header)
  • Flags: 0x02 = SYN
  • Window: 0x07FF = 2047

💻 Most Asked Networking Interview Questions

Placement Focused

💻 Placement Interview Questions & Answers

  1. Q: What is TCP?
    A: Transmission Control Protocol. A connection-oriented, reliable transport layer protocol that provides process-to-process delivery with flow control, error control, and congestion control. It ensures data is delivered in order and without loss.
  2. Q: What is UDP?
    A: User Datagram Protocol. A connectionless, unreliable transport layer protocol. It's lightweight with minimal overhead, suitable for real-time applications (video streaming, VoIP) and simple request-response operations where speed matters more than reliability.
  3. Q: Difference between Hub and Switch?
    A: Hub operates at Physical layer (Layer 1) - broadcasts data to all ports, no intelligence, more collisions. Switch operates at Data Link layer (Layer 2) - learns MAC addresses, forwards selectively to specific ports, reduces collisions, more efficient.
  4. Q: Difference between Router and Gateway?
    A: Router operates at Network layer (Layer 3) - forwards packets based on IP addresses, connects different networks. Gateway can operate at all layers - translates between different protocol stacks (e.g., connect an IP network to an SNA network).
  5. Q: What is ARP?
    A: Address Resolution Protocol. Used to map an IP address (logical) to its corresponding MAC address (physical) on a local network. Operates at the interface between Data Link and Network layers.
  6. Q: What is DNS?
    A: Domain Name System. Hierarchical system that translates human-readable domain names (e.g., www.google.com) to IP addresses. Uses UDP port 53 by default, uses TCP for zone transfers.
  7. Q: What is DHCP?
    A: Dynamic Host Configuration Protocol. Automatically assigns IP addresses, subnet masks, default gateways, and DNS servers to devices on a network. Uses UDP ports 67 (server) and 68 (client).
  8. Q: What is NAT?
    A: Network Address Translation. Technique that allows multiple devices on a private network to share a single public IP address. Conserves IPv4 address space and adds a layer of security by hiding internal IP addresses.
  9. Q: What is Subnetting?
    A: Dividing a large network into smaller subnetworks (subnets) to improve efficiency, security, and management. Uses subnet masks to identify the network and host portions of an IP address.
  10. Q: What is the OSI model?
    A: Open Systems Interconnection model - a 7-layer conceptual framework that standardizes network communication functions: Physical, Data Link, Network, Transport, Session, Presentation, Application.
  11. Q: What is HTTP vs HTTPS?
    A: HTTP (Hypertext Transfer Protocol, port 80) transmits data in plain text. HTTPS (HTTP Secure, port 443) encrypts data using SSL/TLS, providing secure communication.
  12. Q: What is a MAC address?
    A: Media Access Control address - a unique 48-bit (6-byte) hardware identifier assigned to network interface cards (NICs). Written as 12 hexadecimal digits (e.g., A0:B1:C2:D3:E4:F5). Layer 2 addressing.
  13. Q: What is the difference between IPv4 and IPv6?
    A: IPv4 = 32-bit address (~4.3 billion addresses), dotted decimal, has header checksum, supports fragmentation at intermediate routers. IPv6 = 128-bit address (huge address space), hex colon notation, no header checksum, no intermediate fragmentation, built-in security.
  14. Q: What is the Three-Way Handshake in TCP?
    A: Process to establish TCP connection: 1) Client sends SYN, 2) Server responds with SYN+ACK, 3) Client sends ACK. Ensures both sides are ready for communication.
  15. Q: What is the difference between Connection-Oriented and Connectionless service?
    A: Connection-oriented (e.g., TCP) establishes a connection before data transfer, ensures reliable delivery, uses acknowledgments. Connectionless (e.g., UDP) sends data without connection setup, unreliable, no acknowledgments, each packet independent.
  16. Q: What is CSMA/CD?
    A: Carrier Sense Multiple Access with Collision Detection. Used by Ethernet. Devices listen before transmitting. If collision detected, they stop and retry after random backoff time.
  17. Q: What is Firewall?
    A: Security device (hardware or software) that monitors and controls incoming/outgoing network traffic based on predetermined security rules. Can be packet-filter, stateful, or application-layer.
  18. Q: What is Quality of Service (QoS)?
    A: Set of techniques to manage network resources and ensure performance for critical applications. Includes scheduling (FIFO, priority, WFQ), traffic shaping (leaky bucket, token bucket), and resource reservation.
  19. Q: What is the difference between TCP and UDP in terms of header size?
    A: TCP header = 20-60 bytes (with options). UDP header = 8 bytes (fixed).
  20. Q: Explain the concept of encapsulation.
    A: Each layer adds its own header (and trailer) as data moves down the protocol stack. At the receiving end, headers are stripped layer by layer (decapsulation). Ensures each layer gets the information it needs.

Solved Numerical Examples

1. Digital Signal Levels

Q: A digital signal has 8 levels. How many bits per level?
A: log₂8 = 3 bits per level.

2. Download Bit Rate

Q: Download 100 pages/min, each page has 24 lines of 80 chars, each char = 8 bits.
A: 100 × 24 × 80 × 8 = 1,536,000 bps = 1.536 Mbps.

3. Digitized Voice Bit Rate

Q: Voice channel 4 kHz, sample at 2x, each sample = 8 bits. Find bit rate.
A: 4000 × 2 × 8 = 64,000 bps = 64 kbps.

4. Receiver Clock Mismatch

Q: Receiver clock 0.1% faster than sender at 1 kbps and 1 Mbps.
A: At 1 kbps: receives 1001 bps (1 extra). At 1 Mbps: receives 1,001,000 bps (1000 extra).

5. Nyquist Bit Rate

Q: Channel 3000 Hz, 2 signal levels. Max bit rate?
A: 2 × 3000 × log₂2 = 6000 bps. With 4 levels: 2 × 3000 × log₂4 = 12,000 bps.

6. Shannon Capacity

Q: Telephone line 3000 Hz, SNR = 3162. Capacity?
A: 3000 × log₂(1+3162) = 3000 × log₂(3163) ≈ 3000 × 11.62 = 34,860 bps.

7. Attenuation

Q: Power reduced to half. Calculate dB.
A: dB = 10 log₁₀(0.5) = -3 dB (3 dB loss).

8. Mesh Topology

Q: 5 devices in mesh. How many channels and I/O ports?
A: Channels = 5×4/2 = 10. I/O ports per device = 5-1 = 4.

9. IPv4 HLEN Calculation

Q: HLEN = 1000 in binary (8). How many bytes for options?
A: HLEN × 4 = 32 bytes total header. Base = 20 bytes. Options = 12 bytes.

10. Fragment Offset

Q: Offset = 100, HLEN = 5, Total Length = 100. First and last byte?
A: First = 100 × 8 = 800. Header = 20 bytes. Data = 80 bytes. Last = 879.

11. Subnet Calculation

Q: 192.168.1.15/26. Find network ID, broadcast, usable IPs.
A: /26 = 255.255.255.192, block=64, Net=192.168.1.0, BC=192.168.1.63, First=192.168.1.1, Last=192.168.1.62.

12. Baud Rate

Q: Analog signal carries 4 bits/signal element, 1000 signal elements/sec. Find bit rate.
A: N = S × r = 1000 × 4 = 4000 bps.

13. IP Datagram Version/Header Check

Q: IP packet first 8 bits: 0100 0010. Receiver discards. Why?
A: Version=4 (ok), HLEN=2 × 4 = 8 bytes (invalid, min=20). Packet corrupted.

14. Network ID with Default Mask

Q: IP 191.154.25.66. What is network ID with default mask? With /24?
A: Class B default: 191.154.0.0. With /24: 191.154.25.0.

15. Stop-and-Wait Utilization

Q: Bandwidth = 1 Mbps, RTT = 20 ms, Frame = 1000 bits. Utilization?
A: Bandwidth-delay = 1Mbps × 20ms = 20,000 bits. Utilization = 1000/20000 = 5%.
