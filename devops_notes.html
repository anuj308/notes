Cloud Computing & DevOps – Revision Notes

What is DevOps?

DevOps is a set of practices that combines software development (Dev) and IT operations (Ops). Its goal is to shorten the development life cycle and provide continuous delivery of high-quality software.

Key Pillars of DevOps

  1. Continuous Deployment – Automatically deploy every code change that passes automated tests to production.
  2. Automation – Automate repetitive tasks (building, testing, deploying, infrastructure provisioning).
  3. Code Quality – Enforce standards, static analysis, peer reviews, and automated testing.
  4. Monitoring & Observability – Track application performance, logs, metrics, and alerts.
💡 Why DevOps exists – Traditional silos between developers and operations led to slow releases, finger‑pointing, and manual errors.

DevOps vs Old (Traditional) System

Old System (Waterfall / Siloed)

  • Dev writes code → throws it over the wall to Ops.
  • Manual deployment, long release cycles (months).
  • Error‑prone, hard to roll back.

New System (DevOps)

  • Shared responsibility, CI/CD.
  • Infrastructure as Code (IaC).
  • Automated testing, deployment, and monitoring.
  • Faster feedback loops and higher stability.

What is Cloud Computing?

Cloud computing is the on‑demand delivery of IT resources over the internet with pay‑as‑you‑go pricing.

ModelDescription
Public cloudServices offered over the public internet by providers like AWS, Azure, GCP.
Private cloudCloud environment dedicated solely to one organisation.
Hybrid cloudCombination of public + private clouds.
Multi‑cloudUsing two or more public cloud providers simultaneously.

The Relationship Between DevOps and Cloud Computing

AspectHow they work together
AutomationCloud APIs enable scripted provisioning (Terraform, CloudFormation).
Continuous DeploymentCloud provides managed CI/CD services.
MonitoringCloud services feed logs/metrics into DevOps dashboards.
ScalabilityCloud auto‑scaling groups ensure apps handle load.
IaCCloud resources are defined in declarative files.

The SDLC is a structured process used by development teams to design, build, test, deploy, and maintain software.

CI / CD / Continuous Deployment

TermMeaning
CIDevelopers push code frequently. Each push triggers automated build and test.
CD (Delivery)Code is automatically built, tested, and packaged – ready for deployment.
Continuous DeploymentEvery change that passes all tests is automatically deployed to production.

Simple flow: Code change → CI build/test → package artifact → deploy to staging → approval → production deployment → monitoring.

AWS EC2 Instance

EC2 means Elastic Compute Cloud. An EC2 instance is a virtual server in AWS.

SSH Login to EC2

ssh -i key-name.pem ubuntu@public-ip
# For Amazon Linux:
ssh -i key-name.pem ec2-user@public-ip

Install Apache on EC2

sudo apt update
sudo apt install apache2 -y
sudo systemctl start apache2
sudo systemctl enable apache2

AWS RDS - Relational Database Service

RDS is a managed database service for relational databases (MySQL, PostgreSQL, Oracle, SQL Server, Aurora).

Key Features

  • Managed service – AWS handles setup, patching, backups.
  • Multi-AZ – Standby database in another AZ for failover.
  • Read replicas – Extra copies for read traffic.
  • Automated backups – Point-in-time recovery.
