What is Software Testing?

Software testing is the process of checking whether software works correctly and meets requirements. It helps find bugs, missing requirements, wrong output, performance issues, and security vulnerabilities.

Testing Types

TypeDescriptionExamples
Static TestingDone without running the programCode review, walkthrough, inspection, SRS review
Dynamic TestingDone by running the programUnit testing, integration, system testing, acceptance

Verification vs Validation

AspectVerificationValidation
QuestionAre we building the product right?Are we building the right product?
Based onSRS, design documentsUser needs, actual output

Test Levels

  1. Unit/Component Testing — Tests individual functions or modules (done by developers)
  2. Integration Testing — Tests how modules work together
  3. System Testing — Tests the complete application
  4. Acceptance Testing — Validates against user requirements

Alpha vs Beta Testing

AspectAlpha TestingBeta Testing
Done byInternal teamReal users
EnvironmentControlled/companyReal user environment
TimingBefore betaBefore final release
GoalFind major bugsGet real feedback

Key Definitions

  • Test Case — A set of conditions to verify a specific feature
  • Test Suite — Collection of test cases
  • Defect/Bug — An error found during testing
  • Test Plan — Document describing testing strategy
  • Regression Testing — Re-testing after changes to ensure existing features still work
  • Smoke Testing — Quick check to verify basic functionality works
  • Sanity Testing — Focused testing on specific functionality after minor changes
