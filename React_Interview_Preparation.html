1. JavaScript Essentials for React

1.1 Variable Declarations: var, let, const

Featurevarletconst
ScopeFunction scopeBlock scopeBlock scope
Redeclaration✅ Allowed❌ Not in same block❌ Not allowed
Reassignment✅ Allowed✅ Allowed❌ Not allowed
Must Initialize❌ No❌ No (default undefined)✅ Yes
⚠️ Use const by default. Use let only when reassignment is needed. Avoid var entirely.

1.2 Essential Array Methods

.every()     → Check if ALL elements pass a test
const allEven = arr.every(i => i % 2 === 0);

.filter()    → Return elements matching a condition
const evens = arr.filter(i => i % 2 === 0);

.forEach()   → Execute for each element (no return)
arr.forEach(item => console.log(item));

.indexOf()   → Index of value, -1 if not found
const idx = arr.indexOf(3);

Array.from() → Create array from iterable/string
const chars = Array.from('hello'); // ['h','e','l','l','o']

.map()       → Transform each element, return NEW array
const doubled = arr.map(x => x * 2);

.reduce()    → Reduce array to a SINGLE value
const sum = arr.reduce((acc, curr) => acc + curr, 0);

1.3 reduce() – Deep Dive

array.reduce((accumulator, currentValue, currentIndex, array) => {
    // return updated accumulator
}, initialValue)

Key Rules:

  • Accumulator stores accumulated result
  • No initial value provided → first element becomes accumulator
  • Returns single value (number, string, object, array)
  • Does NOT mutate original array
// Sum
[1,2,3,4,5].reduce((a,c) => a + c, 0); // 15

// Find max
[10,5,25,15,20].reduce((a,c) => c > a ? c : a); // 25

// Count occurrences
['a','b','a'].reduce((acc,cur) => {
    acc[cur] = (acc[cur]||0)+1; return acc;
}, {}); // {a:2, b:1}

// Flatten arrays
[[1,2],[3,4]].reduce((a,c) => a.concat(c), []); // [1,2,3,4]

1.4 Destructuring

Array Destructuring

// Basic
const numbers = [10, 20, 30, 40];
const [a, , , b, c = 100] = numbers;
console.log(a); // 10, b: 40, c: 100

// Swap variables
let x = 5, y = 10;
[x, y] = [y, x];

// Nested
const [first, [second, third], fourth] = [1, [2, 3], 4];

Object Destructuring

// Basic
const user = { name: 'John', age: 30, city: 'NY' };
const { name, age, city = 'Unknown' } = user;

// Rename
const { name: n, age: ag } = user;

// In function parameters
function greet({ name, age }) {
    console.log(`${name} is ${age}`);
}
greet(user);

1.5 Spread Operator (...) – Expands iterables

// Merge arrays
const merged = [...[1,2,3], ...[4,5,6]; // [1,2,3,4,5,6]

// Copy array (shallow)
const copy = [...original];

// Spread in function
Math.max(...[5, 10, 15]); // 15

// Merge objects (later overrides earlier)
const settings = { ...defaults, ...prefs };

// Spread string
const chars = [...'hello'];
⚠️ Spread creates SHALLOW copy – nested objects still share references

1.6 Rest Operator (...) – Collects into array

// In function parameters
function sum(...nums) {
    return nums.reduce((a,c) => a + c, 0);
}
sum(1,2,3,4); // 10

// In array destructuring
const [first, ...rest] = [1,2,3,4]; // first=1, rest=[2,3,4]

// In object destructuring
const { name, ...others } = { name:'J', age:30, city:'NY' };

  • JavaScript library for building user interfaces
  • Component-based architecture
  • Declarative (describe WHAT, not HOW)
  • Reusable components
  • Virtual DOM for efficient updates

2.2 JSX (JavaScript XML)

  • Write HTML-like syntax in JavaScript
  • className instead of class
  • htmlFor instead of for
  • Embed JS with { expression }

3. React Hooks – Complete Reference

Hooks can only be called inside function components or custom hooks — and must be called at the top level (not in loops/conditions).

3.1 useState

const [count, setCount] = useState(0);

// Lazy initialization (runs once)
const [data, setData] = useState(() => computeExpensiveInit());

3.2 useEffect

useEffect(() => {
    // Effect logic after render
    console.log('rendered/updated');

    return () => {
        // Cleanup (optional)
        console.log('cleanup');
    };
}, [dep1, dep2]); // Dependencies

Dependency Array Rules

Dependency ArrayBehavior
[]Run ONCE after mount → componentDidMount
[dep]Run after render only when dep changes
No arrayRun after every render
Empty [] ≈ componentDidMount. Cleanup func ≈ componentWillUnmount.

// Access value (without re-render)
nameRef.current.value;

// Persistent mutable value across renders (not in state)
const renderCount = useRef(0);
useEffect(() => { renderCount.current += 1; });

3.5 useReducer

const initialState = { count: 0 };
function reducer(state, action) {
    switch (action.type) {
        case 'INC': return { ...state, count: state.count + 1 };
        case 'DEC': return { ...state, count: state.count - 1 };
        default: return state;
    }
}
const [state, dispatch] = useReducer(reducer, initialState);

// Dispatch
dispatch({ type: 'INC' });
dispatch({ type: 'INC', payload: 5 }); // with payload

3.6 useMemo – Memoize Computed Values

const expensiveValue = useMemo(() => {
    // Heavy calculation — only runs when 'items' changes
    return items.filter(x => x.active).map(x => x.id);
}, [items]); // Dependencies

3.7 useCallback – Memoize Functions

// Without useCallback: new function every render
const handleClick = () => setCount(c => c + 1);

// With useCallback: same reference until deps change
const handleClick = useCallback(() => {
    setCount(c => c + 1);
}, []); // empty deps = never recreated
Use useCallback when passing callbacks to memoized children to prevent unnecessary re-renders.

3.8 Custom Hook Pattern

const useForm = (initialValues, validate, onSubmit) => {
    const [values, setValues] = useState(initialValues);
    const [errors, setErrors] = useState({});
    const [touched, setTouched] = useState({});

    const handleChange = (e) => {
        const { name, value } = e.target;
        setValues(prev => ({ ...prev, [name]: value }));
        if (touched[name]) {
            setErrors(prev => ({ ...prev, [name]: validate({...values, [name]: value})[name] || '' }));
        }
    };

    const handleBlur = (e) => {
        const { name } = e.target;
        setTouched(prev => ({ ...prev, [name]: true }));
        setErrors(validate(values));
    };

    const handleSubmit = async (e) => {
        e.preventDefault();
        const newErrors = validate(values);
        if (Object.values(newErrors).every(e => !e)) {
            await onSubmit(values);
            resetForm();
        } else {
            setErrors(newErrors);
        }
    };

    const resetForm = () => {
        setValues(initialValues);
        setErrors({});
        setTouched({});
    };

    return { values, errors, touched, handleChange, handleBlur, handleSubmit, resetForm };
};

4. Forms – Controlled vs Uncontrolled

4.1 Controlled Form (State-driven)

const [formData, setFormData] = useState({ name: '', email: '' });
const [errors, setErrors] = useState({});

const handleChange = (e) => {
    const { name, value } = e.target;
    setFormData(prev => ({ ...prev, [name]: value }));
};

const validate = () => {
    const errs = {};
    if (!formData.name.trim()) errs.name = 'Name required';
    if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(formData.email))
        errs.email = 'Invalid email';
    return errs;
};

<form onSubmit={handleSubmit}>
    <input name="name"  value={formData.name}  onChange={handleChange} />
    <input name="email" value={formData.email} onChange={handleChange} />
    <button type="submit">Submit</button>
</form>

4.2 Uncontrolled Form (Ref-driven)

const nameRef = useRef(null);
const emailRef = useRef(null);

const handleSubmit = (e) => {
    e.preventDefault();
    const formData = {
        name: nameRef.current.value,
        email: emailRef.current.value
    };
    console.log(formData);
    e.target.reset();
};

<form onSubmit={handleSubmit}>
    <input ref={nameRef} defaultValue="" />
    <input ref={emailRef} defaultValue="" />
    <button type="submit">Submit</button>
</form>

5. State Management (Redux + Context API)

5.1 Redux (Original Pattern)

// store.js
const store = createStore(
    rootReducer,
    window.__REDUX_DEVTOOLS_EXTENSION__ && window.__REDUX_DEVTOOLS_EXTENSION__()
);

// Reducer
function rootReducer(state = initialState, action) {
    switch (action.type) {
        case 'INCREMENT': return { ...state, counter: state.counter + 1 };
        case 'LOGIN':     return { ...state, user: { ...state.user, isLoggedIn: true, name: action.payload } };
        default: return state;
    }
}

5.2 Modern Redux Toolkit (createSlice)

import { createSlice, configureStore } from '@reduxjs/toolkit';

const counterSlice = createSlice({
    name: 'counter',
    initialState: { count: 0 },
    reducers: {
        increment: state => { state.count += 1; },
        decrement: state => { state.count -= 1; },
        incrementByAmount: (state, action) => { state.count += action.payload; }
    }
});

export const { increment, decrement } = counterSlice.actions;
export const store = configureStore({
    reducer: { counter: counterSlice.reducer }
});

5.3 useSelector + useDispatch

import { useSelector, useDispatch } from 'react-redux';
import { increment } from './counterSlice';

function Counter() {
    const count   = useSelector(state => state.counter.count);
    const dispatch = useDispatch();
    return (
        <button onClick={() => dispatch(increment())}>
            Count: {count}
        </button>
    );
}

5.4 Context API

const ThemeContext = createContext(defaultTheme);

function App() {
    const [theme, setTheme] = useState('light');
    return (
        <ThemeContext.Provider value={{ theme, setTheme }}>
            <Toolbar />
        </ThemeContext.Provider>
    );
}

const useTheme = () => useContext(ThemeContext);

function Navbar() {
    const { theme } = useTheme();
    return <nav className={theme}>Navbar</nav>;
}
Redux vs Context: Redux for complex state logic, DevTools, middleware. Context is sufficient for UI-state (theme, user, language).

7. API Integration (Fetch & Axios)

7.1 GET with useEffect

const [data, setData]       = useState([]);
const [loading, setLoading]   = useState(true);
const [error, setError]       = useState(null);

useEffect(() => {
    const fetchData = async () => {
        try {
            const res = await fetch('https://jsonplaceholder.typicode.com/posts');
            if (!res.ok) throw new Error('Failed');
            const json = await res.json();
            setPosts(json.slice(0, 10));
            setError(null);
        } catch (err) {
            setError(err.message);
        } finally {
            setLoading(false);
        }
    };
    fetchData();
}, []); // Empty deps → run once on mount
⚠️ Empty [] prevents infinite fetch loops. Always use it with fetch in useEffect.

9. Performance Optimization

9.1 useMemo & useCallback

const totalExpense = useMemo(() => {
    return expenses.reduce((sum, exp) => sum + exp.amount, 0);
}, [expenses]); // Only recalc when array changes

const addExpense = useCallback((expense) => {
    setExpenses(prev => [...prev, expense]);
}, []); // Function identity stable across renders
useMemo memoizes a VALUE. useCallback memoizes a FUNCTION.

9.2 React.memo – Skip Re-renders

const MemoizedChild = React.memo(function Child({ name }) {
    console.log(`Rendering: ${name}`);
    return <h1>Hello {name}</h1>;
}); // Re-renders only if props change

9.3 Code Splitting (lazy)

import { lazy, Suspense } from 'react';
const HeavyComp = lazy(() => import('./HeavyComp'));

function App() {
    return (
        <Suspense fallback={<div>Loading...</div>}>
            <HeavyComp />
        </Suspense>
    );
}
