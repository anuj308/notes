Framework, MVC & Laravel Introduction

What is a Web Framework?

A web framework is a software platform that provides a structured and standardized way to build web applications.

Popular Web Frameworks

  • Laravel (PHP)
  • Ruby on Rails (Ruby)
  • ASP.NET MVC (C#)
  • Django (Python)
  • Spring (Java)
  • Express (Node.js)
  • Sinatra (Ruby)

What is MVC?

MVC stands for Model-View-Controller — a design pattern that separates application logic into three interconnected components:

  • Model — Represents data and business logic. Interacts with the database.
  • View — Renders the UI. Displays data to the user (Blade templates).
  • Controller — Handles user input, processes requests, and coordinates between Model and View.
Always Remember: MVC is a broader design pattern defining high-level architecture, emphasizing separation of concerns.

What is Laravel?

Laravel is a PHP-based web framework for building high-end web applications using its significant and graceful syntaxes. Developed by Taylor Otwell in July 2011.

Laravel drew inspiration from Ruby on Rails, ASP.NET MVC, and Sinatra.

Key Features of Laravel

🔐 Authentication

Built-in authentication system — just configure models, views, and controllers.

🧩 Blade Templating

Innovative template engine providing clean, expressive syntax with powerful features for dynamic content rendering.

🗄️ Eloquent ORM

Object-Relational Mapping — work with database records as objects using PHP classes and methods.

🔒 Secure Migrations

Version control for database schemas. Structured and consistent database management.

🧪 Unit Testing

Integrates with PHPUnit. Laravel Dusk provides end-to-end browser testing.

🛡️ Security

Bcrypt hashing, CSRF protection, SQL injection prevention.

📦 Libraries & Modules

Pre-installed libraries for authentication, password reset, CSRF protection, etc.

⚡ Artisan CLI

Built-in command-line tool for repetitive tasks — creating models, controllers, migrations, etc.

Installation Requirements

  • XAMPP (supporting PHP > 8.2)
  • Composer — dependency manager for PHP
  • Git
  • VS Code (or any editor)

Installation Commands

# Global Installation (one-time) composer global require laravel/installer laravel new first-app # Per-project Installation composer create-project laravel/laravel first-app

cd first-app php artisan serve # Starts at http://127.0.0.1:8000 (default port 8000)
php artisan serve starts a built-in PHP development server for local testing without needing Apache/Nginx.

php artisan list # Display all Artisan commands

  • Authentication: php artisan make:auth, php artisan breeze:install

  • Queues & Jobs: php artisan queue:work, php artisan make:job
  • Events & Listeners: php artisan make:event, php artisan make:listener

Request Lifecycle & Routing

Laravel Request Lifecycle

A sequence of events that occur during the execution of a request.

🌐 Request → 📄 index.php → ⚙️ bootstrap/app.php → 🔧 Service Providers → 🛣️ Router → 🛡️ Middleware → 🎮 Controller → 🖼️ View → 📨 Response

Detailed Steps

  1. index.php — Entry point. All requests directed here by web server.
  2. Composer Autoloader — Loads all classes automatically (no manual includes).
  3. bootstrap/app.php — Creates Laravel application instance, sets up service container.
  4. Service Providers — Core functionality (mailing, sessions, database, queue, validation, routing). Found in bootstrap/providers.php.
  5. Router — Dispatches request to matching route/controller.
  6. Middleware — Filters HTTP requests (authentication, logging, validation).
  7. Controller — Processes request logic in app/Http/Controllers.
  8. View — Renders Blade template from resources/views.
Default Service Providers: AppServiceProvider, AuthServiceProvider, BroadcastServiceProvider, EventServiceProvider, RouteServiceProvider

Routing Basics

Routing is the process of directing incoming requests (based on URLs) to the appropriate handler code.

Default Route

use Illuminate\Support\Facades\Route; Route::get('/', function () { return view('welcome'); });

HTTP Methods

Route::get($uri, $callback); Route::post($uri, $callback); Route::put($uri, $callback); Route::patch($uri, $callback); Route::delete($uri, $callback); Route::any($uri, $callback); // Any method Route::match(['get', 'post'], $uri, $callback); // Specific methods

Route Parameters

// Required Parameter Route::get('/user/{id}', function ($id) { return "User ID: " . $id; }); // Optional Parameter Route::get('/user/{name?}', function ($name = 'Guest') { return "Hello " . $name; });

Parameter Constraints

// id must be a number Route::get('/user/{id}', function ($id) { return "User ID: " . $id; })->whereNumber('id'); // name must contain only alphabets Route::get('/user/{name}', function ($name) { return "User: " . $name; })->whereAlpha('name');

View Routes

// Directly returns a view Route::view('/welcome', 'welcome'); // View route with data Route::view('/greeting', 'greeting', ['name' => 'John']);

Laravel Responses: Headers, Cookies, JSON

Meaning: A response is what Laravel sends back to the browser/API client after a request. It can be text, HTML view, JSON, redirect, file, header, or cookie.

Where used: APIs return JSON, login redirects users, cookies remember small browser values, custom headers send extra metadata.

Attaching Headers

Route::get('/header-demo', function () { return response('Custom Header Example') ->header('X-App-Name', 'MyLaravelApp') ->header('X-Version', '1.0'); });

Attaching Cookies

Route::get('/response-cookie', function () { return response('Cookie attached') ->cookie('course', 'INT221', 60); });

JSON Response

Route::get('/json-demo', function () { return response()->json([ 'status' => true, 'course' => 'INT221', 'topic' => 'Laravel Response' ]); });

Redirecting to Named Routes and Controller Actions

use App\Http\Controllers\UserController; Route::get('/profile', [UserController::class, 'show'])->name('profile'); Route::get('/go-profile', function () { return redirect()->route('profile'); }); Route::get('/go-controller', function () { return redirect()->action([UserController::class, 'show']); });
Output check: Open /json-demo; browser shows JSON. Open /header-demo; inspect response headers in DevTools Network tab.
Exam line: Laravel response helpers make it easy to return HTML, JSON, redirects, headers, and cookies from the same routing/controller system.

Named Routes

Assign a name to a route so you can generate URLs or redirects using the name instead of the URL path.

Route::get('/user/profile', [UserController::class, 'show']) ->name('profile'); // Using named route $url = route('profile'); return redirect()->route('profile');
Benefit: No need to change <a> tags if the URL changes. Just update the route definition.

Redirection

Send users from one URL/route to another. Common after form submission, login, logout.

// Basic redirect return redirect('home/dashboard'); // Redirect to previous page return redirect()->back(); // Redirect to named route return redirect()->route('dashboard'); // Redirect with flash message return redirect('/home')->with('success', 'Login successful!'); // Redirect with input (keep old form values) return redirect()->back()->withInput(); // Redirect with errors return redirect()->back()->withErrors(['name' => 'Name is required']); // Redirect to external URL return redirect()->away('https://google.com'); // Route shortcut Route::redirect('/old', '/new', 301); // Fallback route (instead of 404) Route::fallback(function () { return "Page not existing"; });

HTTP Status Codes for Redirect

  • 301 — Permanent redirect
  • 302 — Temporary redirect (default)

URL Generation

Generate URLs dynamically using Laravel helper functions instead of hardcoding.

url() Helper

$url = url('/home'); // http://127.0.0.1:8000/home

route() Helper

$url = route('home'); // http://127.0.0.1:8000/home

Asset URLs: asset() Helper

echo asset('css/style.css'); // http://127.0.0.1:8000/css/style.css

action() Helper

$url = action([UserController::class, 'showProfile']);

Current URL Methods

url()->current(); // Current URL url()->previous(); // Previous URL url()->full(); // Full URL with query string

In Blade Templates

// Generation Shortcuts in Blade <a href="{{ url('/home') }}">Home</a> <a href="{{ route('profile') }}">Profile</a> <link rel="stylesheet" href="{{ asset('css/style.css') }}">

Controllers, Middleware & Blade

Controllers

Controllers handle HTTP requests and define application response logic. They group related request handling logic into a single class.

Flow: Route → Controller → View/Response

Creating a Controller

php artisan make:controller StudentController // File: app/Http/Controllers/StudentController.php

Basic Controller Structure

<?php namespace App\Http\Controllers; use Illuminate\Http\Request; class StudentController extends Controller { public function index() { return "All Students"; } }

Route with Controller

use App\Http\Controllers\StudentController; Route::get('/students', [StudentController::class, 'index']);

Passing Data to View

// Controller public function index() { $students = [ ['id' => 1, 'name' => 'Ravi'], ['id' => 2, 'name' => 'Aman'] ]; return view('students', compact('students')); } // View (students.blade.php) @foreach($students as $student) <p>{{ $student['name'] }}</p> @endforeach

Sharing Data with All Views

Use this when the same data is needed on every page, such as app name, logged-in user info, or navigation data.

// app/Providers/AppServiceProvider.php use Illuminate\Support\Facades\View; public function boot(): void { View::share('appName', 'INT221 Laravel Practical'); } // Any Blade file <h1>{{ $appName }}</h1>

Dynamic Route Parameters

// Route Route::get('/student/{id}', [StudentController::class, 'show']); // Controller public function show($id) { // Fetch student by ID return view('student-detail', compact('student')); }

Resource Controller

Auto-generates all CRUD methods with a single command.

php artisan make:controller StudentController --resource

Generated Methods

Method Purpose
index()Show all resources
create()Show create form
store()Save new resource
show()Show single resource
edit()Show edit form
update()Update resource
destroy()Delete resource
// Single route creates ALL CRUD routes Route::resource('students', StudentController::class);

Middleware

Middleware acts as a filter for HTTP requests — inspecting, modifying, allowing, or denying requests before they reach the controller.

📨 Request → 🛡️ Middleware → 🎮 Controller → 📨 Response → 🛡️ Middleware → 🌐 Browser

Creating Middleware

php artisan make:middleware Agecheck php artisan make:middleware Countrycheck

Example: Agecheck (Global Middleware)

<?php namespace App\Http\Middleware; class Agecheck { public function handle(Request $request, Closure $next) { $age = $request->age; if ($age < 18) { return "Not allowed (Age < 18)"; } return $next($request); } }

Registering Middleware

Open bootstrap/app.php:

use Illuminate\Foundation\Configuration\Middleware; ->withMiddleware(function (Middleware $middleware) { // Global (all routes) $middleware->append(\App\Http\Middleware\Agecheck::class); // Route-specific alias $middleware->alias([ 'country.check' => \App\Http\Middleware\Countrycheck::class, ]); });

Using Middleware on Routes

// Single route Route::get('/profile', function () { return "User Profile"; })->middleware('country.check'); // Route group Route::middleware(['country.check'])->group(function () { Route::get('/dashboard', ...); Route::get('/settings', ...); });

Controller Middleware

class AdminController extends Controller { public function __construct() { // Apply to all methods $this->middleware('role'); // Apply to specific methods $this->middleware('role')->only('dashboard'); // Exclude methods $this->middleware('role')->except(['index']); // Multiple middleware $this->middleware(['auth', 'role']); // Middleware with parameter $this->middleware('role:admin'); } }

Template Inheritance

Create a master layout and reuse it across multiple pages (DRY principle).

Step 1: Master Layout

resources/views/layouts/app.blade.php

<!DOCTYPE html> <html> <head> <title>@yield('title')</title> </head> <body> <header><h1>My Website</h1></header> <nav> <a href="/">Home</a> <a href="/about">About</a> </nav> <div> @yield('content') </div> <footer><p>&copy; 2026 My Website</p></footer> </body> </html>

Step 2: Child View

resources/views/home.blade.php

@extends('layouts.app') @section('title') Home Page @endsection @section('content') <h2>Welcome to Home Page</h2> @endsection

Important Directives Summary

Directive Purpose
@extends('layouts.app')Inherit a layout
@section('content') ... @endsectionDefine section content
@yield('content')Display section content (placeholder)
@include('partials.header')Include reusable parts

Secure Routing

Protect routes so only authorized/authenticated users can access them, mainly using auth middleware.

// Single secure route Route::get('/dashboard', function () { return "Welcome to dashboard"; })->middleware('auth'); // Group of secure routes Route::middleware(['auth'])->group(function () { Route::get('/profile', ...); Route::get('/settings', ...); });
Breeze Installation: Install Laravel Breeze for Login/Register pages, auth controllers, and auth middleware. composer require laravel/breeze --dev && php artisan breeze:install && php artisan migrate

Domain Routing

Handle different domains or subdomains in the same Laravel app.

// Basic domain routing Route::domain('admin.localhost')->group(function () { Route::get('/', function () { return "Admin Panel"; }); }); // Dynamic subdomain Route::domain('{account}.localhost')->group(function () { Route::get('/', function ($account) { return "Welcome " . $account; }); });
Add 127.0.0.1 admin.localhost to your hosts file (/etc/hosts on Mac/Linux, C:\Windows\System32\drivers\etc\hosts on Windows).

Route Groups

Apply common properties (middleware, prefix, namespace) to multiple routes.

Route Prefixing

Route prefixing adds the same URL beginning to every route inside a group, such as /admin or /student.

// With prefix Route::prefix('admin')->group(function () { Route::get('/dashboard', ...); // /admin/dashboard Route::get('/users', ...); // /admin/users }); // Combined: prefix + middleware Route::prefix('admin') ->middleware(['auth']) ->group(function () { Route::get('/dashboard', ...); }); // Nested groups Route::prefix('admin')->group(function () { Route::prefix('users')->group(function () { Route::get('/', ...); // /admin/users Route::get('/create', ...); // /admin/users/create }); });

URL Generation, Request Data, Emails, Localization & Sessions

Unit IV practical flow

This unit is about handling data coming from the user/browser and sending data back through URLs, files, cookies, sessions, language files, and emails.

Exam line: Request data is temporary input from the browser; sessions store server-side user data; cookies store small browser-side data; emails and uploads are common real-world request handling tasks.

Request Data Retrieval + Old Input

Meaning: Request data means form fields, query strings, files, headers, and cookies sent by the browser. Laravel gives a Request object to read them safely.

Where used: login forms, search filters, registration forms, profile update forms, file upload forms.

Step-by-step practical

  1. Create a controller.
  2. Create routes to show and submit the form.
  3. Read data using $request->input(), $request->query(), or direct property access.
  4. Use old() in Blade to refill values after validation errors.
php artisan make:controller ProfileController

Routes: routes/web.php

use App\Http\Controllers\ProfileController; Route::get('/profile-form', [ProfileController::class, 'create'])->name('profile.create'); Route::post('/profile-form', [ProfileController::class, 'store'])->name('profile.store'); Route::get('/search', [ProfileController::class, 'search'])->name('profile.search');

Controller

use Illuminate\Http\Request; class ProfileController extends Controller { public function create() { return view('profile_form'); } public function store(Request $request) { $data = $request->validate([ 'name' => 'required|min:3', 'email' => 'required|email' ]); return "Name: " . $data['name'] . ", Email: " . $data['email']; } public function search(Request $request) { $keyword = $request->query('q'); // /search?q=laravel return "Searching for: " . $keyword; } }

Blade view: resources/views/profile_form.blade.php

<form method="POST" action="{{ route('profile.store') }}"> @csrf Name: <input type="text" name="name" value="{{ old('name') }}"> @error('name') <small style="color:red">{{ $message }}</small> @enderror Email: <input type="email" name="email" value="{{ old('email') }}"> @error('email') <small style="color:red">{{ $message }}</small> @enderror <button>Submit</button> </form>
Output check: Open /profile-form. Empty submit shows validation errors. Correct submit prints submitted name and email. Open /search?q=laravel to test query data.
Common mistake: Forgetting @csrf in a POST form causes Laravel to reject the request.

Uploaded Files

Meaning: File upload means receiving a file from an HTML form and storing it in Laravel storage.

Where used: profile photos, assignment uploads, PDF uploads, product images.

Step-by-step practical

  1. Create a form with enctype="multipart/form-data".
  2. Validate the uploaded file.
  3. Store file using $request->file('file')->store().
  4. Run php artisan storage:link if you want browser access to public files.
php artisan make:controller FileController php artisan storage:link

Routes

use App\Http\Controllers\FileController; Route::view('/fileform', 'FileForm')->name('file.form'); Route::post('/upload', [FileController::class, 'uploadData'])->name('file.upload');

Blade form

<form action="{{ route('file.upload') }}" method="POST" enctype="multipart/form-data"> @csrf <input type="file" name="file"> @error('file') <small>{{ $message }}</small> @enderror <button>Upload</button> </form>

Controller based on local tz project pattern

public function uploadData(Request $request) { $request->validate([ 'file' => 'required|file|mimes:jpg,png,pdf|max:2048' ]); $path = $request->file('file')->store('uploads', 'public'); return "File uploaded successfully: " . $path; }
Output check: Uploaded file is stored inside storage/app/public/uploads. Browser URL after storage link: /storage/uploads/filename.ext.
Common mistake: Missing enctype="multipart/form-data" makes $request->file('file') empty.

Cookies

Meaning: A cookie is a small value stored in the browser. Laravel can attach cookies to responses and read them from requests.

Where used: remember theme, remember language, lightweight browser preference.

Set, get, delete cookie

use Illuminate\Http\Request; use Illuminate\Support\Facades\Cookie; Route::get('/cookie/set', function () { return response('Cookie set') ->cookie('student_name', 'Anuj', 60); // minutes }); Route::get('/cookie/get', function (Request $request) { return $request->cookie('student_name', 'No cookie found'); }); Route::get('/cookie/delete', function () { return response('Cookie deleted') ->withoutCookie('student_name'); });
Exam line: Cookies are stored in the browser, so do not store passwords or sensitive data in cookies.

Sending Emails

Meaning: Laravel sends email using Mailables. A Mailable class contains the email subject, data, and Blade email view.

Where used: welcome email, OTP email, contact form, password reset.

Step-by-step practical

  1. Set mail config in .env.
  2. Create a Mailable.
  3. Create email Blade view.
  4. Create form and controller.
  5. Call Mail::to($email)->send(new WelcomeMail(...)).
php artisan make:mail WelcomeMail php artisan make:controller EmailController

.env example for testing

MAIL_MAILER=smtp MAIL_HOST=smtp.gmail.com MAIL_PORT=587 MAIL_USERNAME=your_email@gmail.com MAIL_PASSWORD=your_app_password MAIL_ENCRYPTION=tls MAIL_FROM_ADDRESS=your_email@gmail.com MAIL_FROM_NAME="Laravel App"

Routes and controller based on local tz project

use App\Http\Controllers\EmailController; Route::view('/emailform', 'EmailForm')->name('email.form'); Route::post('/email', [EmailController::class, 'EmailData'])->name('email.send');
use App\Mail\WelcomeMail; use Illuminate\Support\Facades\Mail; public function EmailData(Request $request) { $validated = $request->validate([ 'to' => ['required', 'email'], 'sub' => ['required', 'string', 'max:120'], 'msg' => ['required', 'string', 'max:2000'], ]); Mail::to($validated['to']) ->send(new WelcomeMail($validated['sub'], $validated['msg'])); return back()->with('success', 'Email sent successfully.'); }

Email form

<form method="POST" action="{{ route('email.send') }}"> @csrf <input type="email" name="to" placeholder="Receiver email"> <input type="text" name="sub" placeholder="Subject"> <textarea name="msg" placeholder="Message"></textarea> <button>Send Email</button> </form>
Common mistake: Normal Gmail password will not work. Use an app password or a testing service such as Mailtrap.

Laravel Localization

Meaning: Localization means showing text in different languages using language files from the lang folder.

Where used: multilingual websites, English/Hindi/Punjabi labels, custom validation messages.

Step-by-step practical

  1. Publish/create language files.
  2. Create language folders such as lang/en and lang/hn.
  3. Use __('file.key') in Blade.
  4. Change locale using App::setLocale().
php artisan lang:publish

Language files

// lang/en/welcome.php return [ 'title' => 'Welcome Student', 'message' => 'Laravel practical is ready' ]; // lang/hn/welcome.php return [ 'title' => 'Swagat hai Student', 'message' => 'Laravel practical ready hai' ];

Routes based on local tz localization pattern

Route::get('/lang/en', function () { App::setLocale('en'); return view('Monday'); })->name('lang.en'); Route::get('/lang/hn', function () { App::setLocale('hn'); return view('Monday'); })->name('lang.hn');

Blade

<h1>{{ __('welcome.title') }}</h1> <p>{{ __('welcome.message') }}</p> <a href="{{ route('lang.en') }}">English</a> <a href="{{ route('lang.hn') }}">Hindi</a>
Exam line: Localization separates text from code, so the same view can display different languages.

Laravel Sessions

Meaning: A session stores user-specific data on the server between requests. Laravel stores only a session ID in the browser cookie.

Where used: login state, flash messages, cart data, temporary form success/error messages.

Access, store, delete session data

Route::get('/session/set', function () { session(['student' => 'Anuj']); return 'Session stored'; }); Route::get('/session/get', function () { return session('student', 'No session found'); }); Route::get('/session/delete', function () { session()->forget('student'); return 'Session deleted'; }); Route::get('/session/flash', function () { session()->flash('success', 'Saved successfully'); return redirect('/profile-form'); });

Show flash message in Blade

@if(session('success')) <div class="alert alert-success">{{ session('success') }}</div> @endif
Output check: Open /session/set, then /session/get. Open /session/delete, then /session/get again.
Exam line: Flash session data is available only for the next request, so it is perfect for success and error messages.

Forms, CSRF & Validation

CSRF Protection

Cross-Site Request Forgery protection prevents unauthorized form submissions.

// In Blade form <form method="POST" action="/submit"> @csrf <!-- Generates: <input type="hidden" name="_token" value="..."> --> </form>

Method Field

HTML forms only support GET and POST. Laravel allows PUT, PATCH, DELETE using @method.

<form action="/update" method="POST"> @csrf @method('PUT') </form>

Form Validation

Checking user input before processing.

$validated = $request->validate([ 'name' => 'required|min:3|max:20', 'email' => 'required|email', 'age' => 'required|numeric|min:18|max:60' ]);

Validation Rules

Rule Meaning
requiredField must be filled
emailMust be a valid email
min:3Minimum length 3
max:10Maximum length 10
numericMust be a number

Custom Error Messages

$request->validate([ 'name' => 'required' ], [ 'name.required' => 'Name is mandatory.' ]);

Displaying Errors in Blade

@error('name') <small class="text-danger">{{ $message }}</small> @enderror

Repopulating Forms (Old Input)

Eloquent ORM & Database

Eloquent ORM

Eloquent is Laravel's Object-Relational Mapping (ORM) system. It provides a high-level, object-oriented way to interact with the database.

Benefits

  • Database Abstraction — Switch between MySQL, PostgreSQL, SQLite without changing code
  • Object-Oriented — Models represent database tables as objects
  • Readable Code — Cleaner and more maintainable
  • Portability — Define schema in a database-agnostic way
  • Ease of Development — Reduces complexity of working with SQL
  • Security — Reduces SQL injection vulnerabilities
